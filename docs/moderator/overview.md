# Moderator Role Overview

Welcome to the GOYA admin panel. As a moderator, you have access to a focused set of tools designed to help you review member activity, handle support, and keep the community running smoothly.

## Accessing the Admin Panel

Navigate to `/admin` while logged in. If your account has the `moderator` role, you will land on the admin dashboard automatically. If you see a redirect to the homepage instead, your account has not been granted moderator access — contact an admin.

The admin panel has a collapsible sidebar on the left. Click the arrow icon at the top of the sidebar to expand or collapse it.

---

## What Moderators Can Do

Moderators have access to the following sections of the admin panel:

| Section | What you can do |
|---|---|
| **Dashboard** | View activity overview |
| **Inbox** | Review and action verifications, teacher upgrades, school registrations, events, courses, credits, and support tickets |
| **Users** | Browse and search member profiles |
| **Media** | Browse the media library |
| **Events** | View the full events list |
| **Courses** | View the full courses list |
| **Analytics** | View shop and visitor analytics |
| **Shop** | View orders, products, and coupons |

Your core day-to-day work happens in the **Inbox** — that is where all moderation queues live.

---

## What Only Admins Can Do

The following sections are visible in the sidebar but require admin-level access to use meaningfully. Moderators should not need to interact with these:

- **Settings > System** — Site-wide configuration, maintenance mode, feature flags
- **Settings > Email Templates** — Edit transactional email content
- **Settings > Flows** — Automation flow configuration
- **Settings > Chatbot** — Mattea chatbot configuration
- **Settings > Credits** — Credit type configuration
- **Settings > API Keys** — External API key management
- **Settings > Audit Log** — Full platform audit trail
- **Settings > Migration** — Database migration tooling

If you believe you need access to any of the above, speak with an admin.

## The Sidebar at a Glance

The sidebar organises navigation into top-level links and expandable groups. The **Inbox** link shows a badge with the count of pending school registrations. Inside the Inbox itself, each tab shows its own pending count as an amber badge.

You can collapse the sidebar to icon-only mode by clicking the arrow at the top — useful on smaller screens.

---

## Your Primary Workspace: The Inbox

Everything that needs a moderator decision flows through **Inbox** (`/admin/inbox`). The inbox has seven tabs:

1. **Credits & Hours** — Member credit submissions
2. **Verifications** — Members requesting profile verification
3. **Support Tickets** — Escalated conversations from the Mattea chatbot
4. **Teacher Upgrades** — Members applying to upgrade to teacher status
5. **School Registrations** — Schools awaiting approval
6. **Events** — Member-submitted events awaiting review
7. **Courses** — Member-submitted courses awaiting review

Tabs with pending items show an amber number badge. Start with the **Verifications** and **Events** tabs, as those typically have the most time-sensitive items.
