  <action>
Create the file migration/wp-goya-exporter/wp-goya-exporter.php as a single-file WordPress plugin. The plugin must be structured as follows:

**1. Plugin Header (per WPEX-01):**
```php
<?php
/**
 * Plugin Name: GOYA Exporter
 * Plugin URI: https://goya.com
 * Description: Exports GOYA user data as chunked JSON for migration to v2.
 * Version: 1.0.0
 * Author: GOYA
 * License: GPL v2 or later
 * Text Domain: goya-exporter
 */
```

Add `defined('ABSPATH') || exit;` guard immediately after the header.

**2. Main Plugin Class: `GOYA_Exporter`**

Use a class-based approach wrapping all functionality:

```php
class GOYA_Exporter {
    public function __construct() {
        add_action('admin_menu', [$this, 'register_admin_page']);
        add_action('admin_init', [$this, 'handle_export']);
    }
    // ... methods below
}
new GOYA_Exporter();
```

**3. Admin Page Registration (per WPEX-02):**

Method `register_admin_page()`:
- Call `add_management_page('GOYA Export', 'GOYA Export', 'manage_options', 'goya-export', [$this, 'render_admin_page'])`
- This places it under Tools > GOYA Export

**4. Admin Page Rendering (per WPEX-02, WPEX-10):**

Method `render_admin_page()`:
- Wrap in `<div class="wrap">` with `<h1>GOYA User Export</h1>`
- Form with `method="post"` and `action=""`:
  - **Chunk Size**: `<select name="goya_chunk_size">` with options 100, 250, 500 (default 250)
  - **Offset**: `<input type="number" name="goya_offset" value="0" min="0" step="1">`
  - **Role Filter** (per WPEX-10): `<select name="goya_role_filter">` with options:
    - `all` — "All Users"
    - `subscriber` — "Subscribers Only"
    - `teacher` — "Teachers Only"
    - `wellness` — "Wellness Practitioners Only"
    - `administrator` — "Admins Only"
  - **Status Filter** (per WPEX-10): `<select name="goya_status_filter">` with options:
    - `all` — "All Members"
    - `active` — "Active Members Only" (users with active WooCommerce subscription)
  - **Export Chunk Button**: `<input type="submit" name="goya_export_chunk" value="Export Chunk" class="button button-primary">`
  - **Export All Button**: `<input type="submit" name="goya_export_all" value="Export All" class="button button-secondary">`
  - Add wp_nonce_field('goya_export_nonce', 'goya_nonce')
- Below the form, show total user count: query `count_users()` and display "Total users in database: {N}"
- Add a `<p class="description">` warning for Export All: "Warning: Export All may be slow for 4000+ users. Use chunked export for large databases." (per WPEX-04)

**5. Export Handler (per WPEX-03, WPEX-04, WPEX-05, WPEX-09):**

Method `handle_export()`:
- Check if `$_POST['goya_export_chunk']` or `$_POST['goya_export_all']` is set
- Verify nonce with `wp_verify_nonce($_POST['goya_nonce'], 'goya_export_nonce')`
- Check `current_user_can('manage_options')`
- Read chunk_size, offset, role_filter, status_filter from $_POST

- For Export Chunk: use offset and chunk_size as provided
- For Export All: set offset=0 and number=-1 (all users)

- Build `WP_User_Query` args:
  ```php
  $args = [
      'number' => $number,
      'offset' => $offset,
      'orderby' => 'ID',
      'order' => 'ASC',
  ];
  ```
  If role_filter !== 'all', add `'role' => $role_filter` to args
  (Note: 'wellness' maps to the actual WP role name -- use the value directly as the WordPress role slug)

- Execute query: `$user_query = new WP_User_Query($args)`
- Get results: `$users = $user_query->get_results()`

- For each user, call `$this->build_user_data($user)` to build the export object

- If status_filter === 'active', post-filter to only include users with at least one active subscription (this filtering happens after data enrichment in Plan 02; for now, include a placeholder comment `// TODO: active subscription filter applied after enrichment`)

**6. User Data Builder (per WPEX-05):**

Method `build_user_data(WP_User $user)`:
```php
return [
    'wp_id'        => $user->ID,
    'email'        => $user->user_email,
    'display_name' => $user->display_name,
    'first_name'   => $user->first_name,
    'last_name'    => $user->last_name,
    'role'         => !empty($user->roles) ? $user->roles[0] : 'subscriber',
    'registered_at'=> $user->user_registered,
    'profile'      => [], // Populated by Plan 02 (BuddyBoss xprofile)
    'avatar_url'   => '', // Populated by Plan 02
    'subscriptions'=> [], // Populated by Plan 02 (WooCommerce)
];
```

**7. JSON Output (per WPEX-03, WPEX-09):**

Method `send_json_download(array $users, int $offset, int $limit)`:
- Set headers:
  ```php
  header('Content-Type: application/json; charset=utf-8');
  $filename = ($limit === -1)
      ? 'goya-export-all.json'
      : "goya-export-chunk-{$offset}-{$limit}.json";
  header("Content-Disposition: attachment; filename={$filename}");
  ```
- `echo json_encode($users, JSON_PRETTY_PRINT | JSON_UNESCAPED_UNICODE);`
- `exit;`

**Read-only guarantee (per WPEX-09):** The entire plugin uses ONLY SELECT queries and WP read functions. No insert, update, delete, or write operations anywhere. Add a comment at the top of the class: `// READ-ONLY: This plugin performs zero writes to the WordPress database.`
