  <action>
**Install the Google Analytics Data API package:**
```bash
npm install @google-analytics/data
```

**Create `lib/analytics/ga4.ts`** with:

1. **`getGA4PropertyId()`** — async function that queries `site_settings` table via `getSupabaseService()` for key `ga4_property_id`, returns the value string or null if not set.

2. **`getGA4Client()`** — creates a `BetaAnalyticsDataClient` from `@google-analytics/data`. Auth via service account:
   - Read `GOOGLE_SERVICE_ACCOUNT_KEY` env var (JSON string of service account credentials)
   - Parse the JSON and pass as `credentials` option: `{ credentials: { client_email, private_key } }`
   - If env var missing, return null

3. **`runGA4Report(options)`** — main exported function. Parameters:
   ```typescript
   interface GA4ReportOptions {
     propertyId: string
     startDate: string    // YYYY-MM-DD or 'NdaysAgo' format
     endDate: string      // YYYY-MM-DD or 'today'
     metrics: string[]    // e.g. ['sessions', 'screenPageViews']
     dimensions?: string[] // e.g. ['date', 'country']
     orderBys?: { metric?: string; dimension?: string; desc?: boolean }[]
     limit?: number
   }
   ```
   Returns: `{ rows: { dimensions: string[], metrics: number[] }[] } | null`

   Implementation:
   - Get client via `getGA4Client()`, return null if no client
   - Call `client.runReport()` with the options mapped to GA4 API format
   - Property format: `properties/${propertyId}`
   - Map response rows to simplified format: each row's dimensionValues as string array, metricValues as number array
   - Wrap in try/catch, log errors, return null on failure

4. **`isGA4Configured()`** — async function, returns true if both `getGA4PropertyId()` returns non-null AND `GOOGLE_SERVICE_ACCOUNT_KEY` env var exists. Used by the page to decide whether to show fallback UI.

Export all four functions. Per VIS-01.

**Replace `app/admin/analytics/visitors/page.tsx`** with a server component:

1. **Check GA4 configuration** — call `isGA4Configured()`. If false, render the fallback UI (per VIS-10):
   - Centered card with icon, "GA4 Not Configured" heading
   - Instructions: "Set the `ga4_property_id` in Site Settings and add the `GOOGLE_SERVICE_ACCOUNT_KEY` environment variable to enable visitor analytics."
   - Return early — do not attempt any GA4 API calls.

2. **Accept searchParams** for time range — `range` param with values: '7d', '30d', '90d', '6m'. Default '30d'.

3. **Compute date ranges** from the range param:
   - Current period: startDate/endDate for the selected range
   - Previous period: same duration immediately before startDate (for trend calculation per VIS-03)
   - Use GA4 date format 'YYYY-MM-DD' or relative like 'NdaysAgo'

4. **Fetch GA4 data** using `runGA4Report` (parallel with Promise.all):

   a. **Overview metrics** (current period) — metrics: sessions, screenPageViews, totalUsers, bounceRate, averageSessionDuration, newUsers. No dimensions.

   b. **Overview metrics** (previous period) — same metrics, previous date range. For trend calculation.

   c. **Sessions by date** (current period) — metrics: sessions, dimensions: date. For the traffic chart (VIS-05).

   d. **Top pages** — metrics: sessions, screenPageViews, averageSessionDuration. Dimensions: pagePath. Limit 10, order by sessions desc (VIS-06).

   e. **Traffic sources** — metrics: sessions. Dimensions: sessionSourceMedium. Limit 10, order by sessions desc (VIS-07).

   f. **Countries** — metrics: sessions. Dimensions: country. Limit 10, order by sessions desc (VIS-08).

   g. **Devices** — metrics: sessions. Dimensions: deviceCategory. No limit (VIS-09).

5. **Compute trends** for each overview metric: `((current - previous) / previous * 100)`. Positive = up, negative = down. Format as "+12.3%" or "-5.1%".

6. **Compute percentages** for sources, countries, devices: each row's sessions / total sessions * 100.

7. **Render layout**:
   - Title "Visitor Analytics"
   - Pass all computed data to `<VisitorsAnalyticsClient />` as props (serializable data only)
