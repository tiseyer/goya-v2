Refund pattern (ORD-05):
```typescript
// Full refund
await stripe.refunds.create({ payment_intent: paymentIntentId })
// Partial refund
await stripe.refunds.create({ payment_intent: paymentIntentId, amount: amountCents })
```

Cancel subscription pattern (ORD-06):
```typescript
// Schedule cancellation (cancel at period end)
await stripe.subscriptions.update(subscriptionId, { cancel_at_period_end: true })
// Immediate cancellation
await stripe.subscriptions.cancel(subscriptionId)
```

Timeline assembly (ORD-07):
```typescript
// Filter webhook_events by event_type patterns + created_at window
const { data: events } = await supabase
  .from('webhook_events')
  .select('event_type, created_at, payload, status')
  .or(`event_type.like.%payment_intent%,event_type.like.%subscription%,event_type.like.%invoice%,event_type.like.%charge%`)
  .order('created_at', { ascending: true })
// Then filter in JS for events whose payload references the order's stripe_id
```

Invoice pattern (ORD-09):
```typescript
await stripe.invoices.sendInvoice(invoiceId) // resend
const invoice = await stripe.invoices.retrieve(invoiceId) // get PDF URL: invoice.invoice_pdf
```

`webhook_events` table:
- id (uuid), stripe_event_id (text UNIQUE), event_type (text), payload (jsonb), status (text), created_at (timestamptz)

`stripe_orders` table:
- stripe_id (the payment_intent ID or subscription ID)
- stripe_customer_id, user_id, amount_total, status, type, subscription_status, cancel_at_period_end, current_period_start, current_period_end, canceled_at

`profiles` table:
- id, email, full_name, stripe_customer_id, role, avatar_url
