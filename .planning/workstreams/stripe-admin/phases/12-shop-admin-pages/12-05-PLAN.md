`stripe_coupons` table (current columns):
- id (uuid PK), stripe_coupon_id (text UNIQUE), stripe_promotion_code_id (text UNIQUE nullable), name (text), code (text nullable), discount_type (percent|amount|free_product), percent_off (numeric 5,2), amount_off (integer), currency (text), duration (forever|once|repeating), duration_in_months (integer), max_redemptions (integer), times_redeemed (integer), redeem_by (timestamptz), valid (boolean), metadata (jsonb), created_at (timestamptz), updated_at (timestamptz)

**NEW columns added by this plan's migration (CPN-02):**
- role_restrictions (jsonb DEFAULT '{}') — Stores `{ mode: 'whitelist'|'blacklist', roles: string[] }`. GOYA-local only, NOT sent to Stripe.
- product_restrictions (jsonb DEFAULT '{}') — Stores `{ mode: 'whitelist'|'blacklist', product_ids: string[] }`. GOYA-local only, NOT sent to Stripe.

Stripe Coupon + Promotion Code pattern (from research):
```typescript
// Create coupon
const coupon = await stripe.coupons.create({ name, percent_off?, amount_off?, currency?, duration, max_redemptions?, redeem_by? })
// If public code provided, create promotion code
if (publicCode) {
  const promoCode = await stripe.promotionCodes.create({ coupon: coupon.id, code: publicCode })
  // Store promoCode.id as stripe_promotion_code_id
}
```

Pitfall 6: free_product type = 100% off coupon in Stripe. Store discount_type: 'free_product' locally for display, but send percent_off: 100 to Stripe.

Stripe coupons.update() limited fields: name, metadata. Cannot change percent_off/amount_off after creation.
