2. **CREDITS_SORT_FIELDS** export: `['created_at', 'updated_at', 'activity_date', 'amount', 'credit_type', 'status']` (matches the existing CREDITS_SORT_FIELDS in users.ts but as the canonical export for the credits routes).

3. **ListCreditsParams** interface:
   - `pagination: PaginationParams`
   - `status?: 'pending' | 'approved' | 'rejected'`
   - `user_id?: string`
   - `credit_type?: CreditType`
   - `date_from?: string`
   - `date_to?: string`

4. **listCredits(params):** Query `credit_entries` with `select('*', { count: 'exact' })`. Apply optional `.eq('status', status)`, `.eq('user_id', user_id)`, `.eq('credit_type', credit_type)`, `.gte('activity_date', date_from)`, `.lte('activity_date', date_to)` filters. Apply `.order()` and `.range()` from pagination. Return `{ data, count, error }`.

5. **getCreditById(id):** Query `credit_entries` with `.eq('id', id).single()`. Return `{ data, error }`.

6. **CreateCreditParams** interface: `{ user_id: string, credit_type: CreditType, amount: number, activity_date: string, description?: string | null, source?: 'manual' | 'automatic', status?: 'pending' | 'approved' | 'rejected' }`.

7. **createCredit(params):** Insert into `credit_entries` with `.insert(params).select().single()`. Return `{ data, error }`.

8. **ALLOWED_CREDIT_UPDATE_FIELDS** export: `['status', 'rejection_reason']` (only status changes and rejection reason via PATCH).

9. **UpdateCreditParams** interface: `{ status?: 'pending' | 'approved' | 'rejected', rejection_reason?: string | null }`.

10. **updateCredit(id, updates):** Validate keys against ALLOWED_CREDIT_UPDATE_FIELDS (same pattern as events.ts updateEvent). Update `credit_entries` with `.update(updates).eq('id', id).select().single()`. Return `{ data, error }`.

11. **getCreditSummary(userId):** Query `credit_entries` where `user_id = userId`, `status = 'approved'`, and `expires_at >= today` (ISO date string). Select `credit_type, amount`. Aggregate in JS into `{ ce: number, karma: number, practice: number, teaching: number, community: number, total: number }`. Return `{ data: summary, error }`. Use the same logic as `getUserCreditTotals` in `lib/credits.ts` but adapted for service-role client pattern.

All functions use `getSupabaseService() as any` with the eslint-disable comment, consistent with all other service files.
