  </read_first>
  <action>
    Create `lib/api/services/analytics.ts` with three exported async functions. Follow the established service pattern (getSupabaseService() as any).

    **getOverviewMetrics()** — per ANLY-01:
    - Query `profiles` table: `select('id, role, member_type, created_at, onboarding_completed, subscription_status')`
    - Compute: `total_members` = count of all profiles, `active_members` = profiles where subscription_status='member', `new_this_month` = profiles where created_at >= first day of current month (use `new Date()` to get current year/month, build ISO date string `YYYY-MM-01`)
    - Return object: `{ total_members: number, active_members: number, new_this_month: number }`
    - On query error, return `{ data: null, error }`

    **getMembershipStats()** — per ANLY-02:
    - Accept optional `date_from` and `date_to` string params (ISO dates). Default: last 30 days (use `new Date()` math, subtract 30*24*60*60*1000).
    - Query in parallel via Promise.all: profiles (same select as overview), stripe_orders (`select('id, stripe_id, stripe_customer_id, stripe_price_id, stripe_product_id, user_id, amount_total, type, subscription_status, cancel_at_period_end, canceled_at, created_at')`), schools (`select('owner_id')`)
    - Map query results to match the `ProfileRow[]`, `OrderRow[]` types from lib/analytics/metrics.ts (same mapping pattern as app/admin/shop/analytics/page.tsx lines 89-123)
    - Build schoolOwnerIds Set from schools data
    - Call `computeFunnelMetrics(profiles, orders, from, to, 'all', schoolOwnerIds)` from lib/analytics/metrics.ts
    - Return the FunnelMetrics object as data
    - No Stripe API calls — all from local Supabase tables

    **getRevenueStats()** — per ANLY-03:
    - Accept optional `date_from` and `date_to` string params (same defaults as memberships)
    - Query in parallel: stripe_orders (same select), stripe_prices (`select('stripe_id, unit_amount, type, interval, interval_count')`), profiles (same select), schools (`select('owner_id')`)
    - Map results to match `OrderRow[]`, `PriceRow[]`, `ProfileRow[]` types
    - Build profileMap (Map of id->profile) and schoolOwnerIds Set
    - Call `computeRevenueMetrics(orders, prices, from, to, 'all', profileMap, schoolOwnerIds)` from lib/analytics/metrics.ts
    - Also call `bucketTimeSeries(orders, from, to, granularity)` where granularity = 'daily' if range <= 60 days, else 'weekly'
    - Return `{ ...revenueMetrics, time_series: chartData }` as data
    - No Stripe API calls

    Each function returns `{ data, error }` pattern consistent with other services.
  </action>
