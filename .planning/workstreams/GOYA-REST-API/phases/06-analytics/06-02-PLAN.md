  <action>
    Create `lib/api/services/analytics-engagement.ts` with two exported async functions. Follow established service pattern (getSupabaseService() as any).

    **getEngagementStats(params?: { date_from?: string; date_to?: string })** — per ANLY-04:
    Query in parallel via Promise.all:
    1. `events` table: `select('id, status, start_date, max_spots, deleted_at, created_at')` filtered by `.is('deleted_at', null)` (consistent with events service soft-delete pattern). If date_from provided, filter `.gte('start_date', date_from)`. If date_to, filter `.lte('start_date', date_to)`.
    2. `event_registrations` table: `select('id, event_id, status, created_at')`.
    3. `courses` table: `select('id, status, deleted_at, created_at')` filtered by `.is('deleted_at', null)`. Apply same date filters on created_at.
    4. `course_enrollments` table: `select('id, course_id, user_id, status, completed_at, created_at')`.

    Compute and return:
    ```
    {
      events: {
        total_events: number,           // count of non-deleted events (with date filter)
        total_registrations: number,    // count of registrations for those events
        avg_registrations_per_event: number,  // total_registrations / total_events (0 if no events)
      },
      courses: {
        total_courses: number,          // count of non-deleted courses (with date filter)
        total_enrollments: number,      // count of enrollments for those courses
        completed_enrollments: number,  // enrollments where status = 'completed'
        completion_rate: number,        // completed / total_enrollments (0 if no enrollments)
      }
    }
    ```

    For registrations/enrollments counts: filter by event_ids/course_ids that are in the events/courses result set (to respect soft-delete and date filters). Build a Set of event/course IDs from the first queries, then filter registration/enrollment rows by that Set.

    **getCreditStats(params?: { date_from?: string; date_to?: string })** — per ANLY-05:
    Query `credit_entries` table: `select('id, credit_type, amount, status, activity_date, created_at')`. If date_from, filter `.gte('activity_date', date_from)`. If date_to, filter `.lte('activity_date', date_to)`.

    Compute and return:
    ```
    {
      total_submissions: number,        // total count
      by_status: {
        pending: number,
        approved: number,
        rejected: number,
      },
      by_type: Record<string, { count: number, total_hours: number }>,  // keyed by credit_type
      total_approved_hours: number,     // sum of amount where status='approved'
    }
    ```

    Aggregate in JS (same pattern as getCreditSummary in credits.ts). Loop over data rows, increment counters for by_status based on row.status, build by_type map keyed by credit_type with count and sum of amount, and track total_approved_hours.

    Each function returns `{ data, error }` consistent with other services.
  </action>
