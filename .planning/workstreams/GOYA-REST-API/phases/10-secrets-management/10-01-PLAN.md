    1. Create migration `supabase/migrations/20260351_admin_secrets.sql` (per SECR-08):
       - Table `admin_secrets` with columns: id (uuid PK default gen_random_uuid()), key_name (text NOT NULL UNIQUE), encrypted_value (text NOT NULL), iv (text NOT NULL), description (text DEFAULT ''), category (text NOT NULL DEFAULT 'Other'), created_at (timestamptz NOT NULL DEFAULT now()), updated_at (timestamptz NOT NULL DEFAULT now()), created_by (uuid REFERENCES profiles(id) ON DELETE SET NULL)
       - Add index on category: `CREATE INDEX idx_admin_secrets_category ON admin_secrets (category);`
       - Enable RLS with no policies (same as api_keys): `ALTER TABLE admin_secrets ENABLE ROW LEVEL SECURITY;`
       - Apply migration via `supabase db query --linked` (same pattern as Phase 03/04 — avoids timestamp collision issues with db push)

    2. Create encryption service `lib/secrets/encryption.ts` (per SECR-07):
       - Import `crypto` from Node.js (built-in, no npm install)
       - `const ALGORITHM = 'aes-256-gcm'`
       - Read SECRETS_MASTER_KEY from `process.env.SECRETS_MASTER_KEY`; throw Error if missing at call time
       - Derive 32-byte key from SECRETS_MASTER_KEY using `crypto.createHash('sha256').update(masterKey).digest()` (ensures exactly 32 bytes regardless of input format)
       - `export function encrypt(plaintext: string): { encrypted: string; iv: string }` — generates random 16-byte IV via `crypto.randomBytes(16)`, creates cipher with `crypto.createCipheriv(ALGORITHM, key, iv)`, concatenates cipher.update + cipher.final + authTag (16 bytes), returns base64-encoded encrypted and iv
       - `export function decrypt(encrypted: string, iv: string): string` — creates decipher with `crypto.createDecipheriv(ALGORITHM, key, ivBuffer)`, extracts authTag from last 16 bytes of encrypted buffer, sets auth tag, deciphers remainder, returns plaintext string
       - Both functions are synchronous, server-side only
       - Add `import 'server-only'` at top to prevent accidental client import

    3. Update `.env.local.example` (per SECR-10):
       - Add section after existing vars:
         ```

         # Generate with: openssl rand -base64 32
         SECRETS_MASTER_KEY=your-secrets-master-key
         ```

    Five server actions:

    1. `export async function listSecrets(): Promise<{ success: true; secrets: SecretListItem[] } | { success: false; error: string }>` (per SECR-01, SECR-09)
       - Query admin_secrets, select ONLY `id, key_name, description, category, updated_at` — NEVER select encrypted_value or iv in list
       - Order by updated_at desc
       - Return SecretListItem[] — no decrypted values

    2. `export async function createSecret(name: string, value: string, category: SecretCategory, description: string): Promise<{ success: true } | { success: false; error: string }>` (per SECR-02)
       - Validate: name must be non-empty trimmed string, value must be non-empty
       - Get current user via createSupabaseServerActionClient
       - Call encrypt(value) from lib/secrets/encryption
       - Insert into admin_secrets: key_name=name.trim(), encrypted_value, iv, category, description=description.trim(), created_by=user?.id
       - revalidatePath('/admin/api-keys')

    3. `export async function getSecret(id: string): Promise<{ success: true; secret: SecretListItem & { value: string } } | { success: false; error: string }>` (per SECR-09)

       - Select full row including encrypted_value and iv
       - Call decrypt(encrypted_value, iv)
       - Return SecretListItem fields + decrypted value

    4. `export async function updateSecret(id: string, fields: { key_name?: string; value?: string; category?: SecretCategory; description?: string }): Promise<{ success: true } | { success: false; error: string }>` (per SECR-03)
       - UUID validation
       - Build update object: if fields.key_name, set key_name=fields.key_name.trim(); if fields.value, call encrypt(fields.value) and set encrypted_value+iv; if fields.category, set category; if fields.description, set description=fields.description.trim()
       - Always set updated_at to new Date().toISOString()
       - revalidatePath('/admin/api-keys')

    5. `export async function deleteSecret(id: string): Promise<{ success: true } | { success: false; error: string }>` (per SECR-04)
       - UUID validation
       - Hard delete from admin_secrets (not soft delete — secrets have no audit trail per deferred SECR-F02)
       - revalidatePath('/admin/api-keys')
