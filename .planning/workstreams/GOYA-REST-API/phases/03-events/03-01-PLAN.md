**`export async function listEvents(params: ListEventsParams)`** -- per EVNT-01:

- Build query: `supabase.from('events').select('*', { count: 'exact' })`

- Apply filters conditionally:
  - `.eq('category', category)` if provided
  - `.eq('status', status)` if provided
  - `.eq('format', format)` if provided
  - `.gte('date', date_from)` if provided
  - `.lte('date', date_to)` if provided
- Apply sort and range from pagination (same pattern as listUsers)
- Return `{ data, count, error }`

**`export async function getEventById(id: string)`** -- per EVNT-02:


Validation constants:
```typescript
const VALID_CATEGORIES: EventCategory[] = ['Workshop', 'Teacher Training', 'Dharma Talk', 'Conference', 'Yoga Sequence', 'Music Playlist', 'Research'];
const VALID_FORMATS: EventFormat[] = ['Online', 'In Person', 'Hybrid'];
const VALID_STATUSES: EventStatus[] = ['published', 'draft', 'cancelled', 'deleted'];
```

**GET handler** (EVNT-01):
1. Auth + rate limit + requirePermission('read') -- same 3-step pattern as users
2. Parse pagination: `parsePaginationParams(ctx.url, EVENTS_SORT_FIELDS)`
3. Extract filter params from ctx.url.searchParams:
   - `category` -- validate against VALID_CATEGORIES, ignore if invalid
   - `status` -- validate against VALID_STATUSES, ignore if invalid
   - `format` -- validate against VALID_FORMATS, ignore if invalid
   - `date_from` -- pass through as string
   - `date_to` -- pass through as string
4. Call `listEvents({ pagination, category, status, format, date_from, date_to })`
5. If error: `errorResponse('QUERY_ERROR', 'Failed to fetch events', 500)`
6. Build pagination meta, return `paginatedResponse(data, meta)`

**POST handler** (EVNT-03):
1. Auth + rate limit + requirePermission('write')
2. Parse JSON body (try/catch, 400 on failure)
3. Validate required fields: `title` (non-empty string), `category` (in VALID_CATEGORIES), `format` (in VALID_FORMATS), `date` (non-empty string), `time_start` (non-empty string), `time_end` (non-empty string). Return 400 with specific error message for each missing/invalid required field.
4. Validate optional enum fields if present: `status` against VALID_STATUSES
5. Build CreateEventParams from validated body -- only pass through known fields
6. Call `createEvent(params)`
7. If error: `errorResponse('CREATE_ERROR', 'Failed to create event', 500)`
8. Audit log: `await ctx.logAudit({ category: 'admin', action: 'event.create', target_type: 'event', target_id: data.id, description: 'Created event via API', metadata: { title: data.title } })`
9. Return `successResponse(data, 201)`

Export: `export const GET = handlers.GET; export const POST = handlers.POST;`

5. Validate enum fields if present:
   - `category` against VALID_CATEGORIES
   - `format` against VALID_FORMATS
