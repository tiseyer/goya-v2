  <action>
    Create `app/admin/api-keys/actions.ts` with 'use server' and 'server-only':

    1. `createApiKey(name: string, permissions: ApiKeyPermission[])` server action:
       - Generate a random 32-byte hex key using `crypto.randomBytes(32).toString('hex')`
       - Compute key_prefix as first 8 chars of the raw key
       - Hash with SHA-256 (same as middleware.ts pattern: `createHash('sha256').update(rawKey).digest('hex')`)
       - Get current user via `createSupabaseServerClient()` then `supabase.auth.getUser()` for created_by
       - Insert into api_keys via `getSupabaseService() as any` (table not in generated types — same pattern as all other phases)
       - Call revalidatePath('/admin/api-keys')
       - Return `{ success: true, rawKey }` — this is the ONLY time the raw key is available
       - On error return `{ success: false, error: string }`

    2. `revokeApiKey(keyId: string)` server action:
       - Update api_keys set active=false where id=keyId via getSupabaseService() as any
       - Call revalidatePath('/admin/api-keys')
       - Return `{ success: true }` or `{ success: false, error: string }`

    Create `app/admin/api-keys/page.tsx` as async server component:
    - Query api_keys table via getSupabaseService() as any, select all columns, order by created_at desc
    - Cast result to ApiKeyRow[]
    - Render page header "API Keys" with count display (same style as products page: text-2xl font-bold text-[#1B3A5C])
    - Pass keys data to `<ApiKeysTable initialKeys={keys} />`
    - Handle DB error with red error box (same pattern as products page)
  </action>
