**getSettingByKey(key: string)** — Per ADMN-03
- Query `site_settings` with `.select('*').eq('key', key).single()`.
- Return `{ data, error }`.

**updateSettings(updates: Record<string, string>)** — Per ADMN-02
- Accepts an object like `{ "ga4_measurement_id": "G-ABC123", "analytics_enabled": "true" }`.
- For each key-value pair, upsert into `site_settings`:
  - Use a loop: for each `[key, value]` in `Object.entries(updates)`, call `.update({ value, updated_at: new Date().toISOString() }).eq('key', key)`.
- After all updates, call `getAllSettings()` to return the full updated state.
- Return `{ data, error }`.
- If any individual update fails, return the first error.

**updateSettingByKey(key: string, value: string)** — Per ADMN-04
- Update `site_settings` with `.update({ value, updated_at: new Date().toISOString() }).eq('key', key).select().single()`.
- Return `{ data, error }`.

Use `getSupabaseService() as any` pattern.

Create two route files. CRITICAL: all four handlers use `requirePermission(key, 'admin')` — these are admin-only endpoints.

**app/api/v1/admin/settings/route.ts** — GET (ADMN-01) + PATCH (ADMN-02):

GET handler:
- Auth: `validateApiKey` -> `rateLimit` -> `requirePermission(key, 'admin')`.
- Call `getAllSettings()`.
- If error, return `errorResponse('QUERY_ERROR', 'Failed to fetch settings', 500)`.
- Return `successResponse(data)`.

PATCH handler (bulk update):
- Auth: `validateApiKey` -> `rateLimit` -> `requirePermission(key, 'admin')`.
- Parse JSON body. Expect `{ settings: { "key1": "value1", "key2": "value2" } }`.
- Validate: `body.settings` must be a non-null object with at least one key.
- Validate: all values must be strings (site_settings.value is text type).
- Call `updateSettings(body.settings)`.
- If error, return 500.
- Audit: `{ category: 'admin', action: 'settings.bulk_update', target_type: 'site_settings', description: 'Bulk updated admin settings via API', metadata: { keys: Object.keys(body.settings) } }`.
- Return `successResponse(data)`.

**app/api/v1/admin/settings/[key]/route.ts** — GET (ADMN-03) + PATCH (ADMN-04):

Extract `key` from URL: last segment of pathname.
Validate key: must be non-empty, only alphanumeric + underscores (regex `/^[a-z0-9_]+$/`). Return 400 if invalid.

GET handler:
- Auth: `validateApiKey` -> `rateLimit` -> `requirePermission(key, 'admin')`. (Note: `key` the URL param vs `key` the API key — use distinct variable names: `settingKey` for URL param, `apiKey` for auth result.)
- Call `getSettingByKey(settingKey)`.
- If error or !data, return `errorResponse('NOT_FOUND', 'Setting not found', 404)`.
- Return `successResponse(data)`.

PATCH handler:
- Auth: `validateApiKey` -> `rateLimit` -> `requirePermission(apiKeyData, 'admin')`.
- Parse JSON body. Expect `{ value: "new_value" }`.
- Validate: `body.value` must be a string (can be empty string).
- Call `updateSettingByKey(settingKey, body.value)`.
- If error or !data: return `errorResponse('NOT_FOUND', 'Setting not found', 404)`.
- Audit: `{ category: 'admin', action: 'settings.update', target_type: 'site_settings', target_id: data.id, description: 'Updated admin setting via API', metadata: { key: settingKey } }`.
- Return `successResponse(data)`.
