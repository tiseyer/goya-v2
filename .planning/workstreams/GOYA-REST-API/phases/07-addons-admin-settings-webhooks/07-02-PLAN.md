export async function getUserCertifications(userId: string) {

  <action>
Append three new functions to the existing `lib/api/services/addons.ts` file (created in Plan 01).

**getUserAddons(userId: string)** — Per ADON-06
- Query `user_designations` table.
- `.select('*, products(name, full_name, category, slug)')` — join with products table for product info.
- Filter `.eq('user_id', userId).is('deleted_at', null)` — only active assignments.
- This is the same query as `getUserCertifications` in users.ts but scoped to the addons service for the addons API path.
- Return `{ data, error }`.

**assignAddonToUser(userId: string, params: AssignAddonParams)** — Per ADON-07
- Interface `AssignAddonParams`: `{ stripe_product_id: string; stripe_price_id: string }`.
- First check if assignment already exists: query `user_designations` for `.eq('user_id', userId).eq('stripe_product_id', params.stripe_product_id).is('deleted_at', null).maybeSingle()`.
- If existing row found, return `{ data: null, error: 'ALREADY_ASSIGNED' as const }`.
- Insert new row: `{ user_id: userId, stripe_product_id: params.stripe_product_id, stripe_price_id: params.stripe_price_id }`.
- `.select().single()`.
- Return `{ data, error }`.

**removeAddonFromUser(userId: string, addonId: string)** — Per ADON-08
- addonId here is the `user_designations.id` (the assignment row ID, NOT the product ID).
- Soft-delete: `.update({ deleted_at: new Date().toISOString() }).eq('id', addonId).eq('user_id', userId).is('deleted_at', null).select().single()`.
- Return `{ data, error }`.

Create two route files following the event registrations pattern.

**app/api/v1/addons/users/[userId]/route.ts** — GET (ADON-06) + POST (ADON-07):

Extract `userId` from URL: `const segments = ctx.url.pathname.split('/'); const userId = segments[segments.indexOf('users') + 1];`
Validate UUID format — return 400 if invalid.

GET handler:
- Auth: `validateApiKey` -> `rateLimit` -> `requirePermission(key, 'read')`.
- Call `getUserAddons(userId)`.
- If error, return `errorResponse('QUERY_ERROR', 'Failed to fetch user add-ons', 500)`.
- Return `successResponse(data)` (no pagination — user typically has few designations).

POST handler:
- Auth: `validateApiKey` -> `rateLimit` -> `requirePermission(key, 'write')`.
- Parse JSON body.
- Validate required fields: `stripe_product_id` (non-empty string), `stripe_price_id` (non-empty string).
- Call `assignAddonToUser(userId, { stripe_product_id, stripe_price_id })`.
- If error === 'ALREADY_ASSIGNED', return `errorResponse('ALREADY_ASSIGNED', 'User already has this add-on', 409)`.
- If other error, return 500.
- Audit: `{ category: 'admin', action: 'addon.assign', target_type: 'user_designation', target_id: data.id, description: 'Assigned add-on to user via API', metadata: { user_id: userId, stripe_product_id } }`.
- Return `successResponse(data, 201)`.

**app/api/v1/addons/users/[userId]/[addonId]/route.ts** — DELETE (ADON-08):

Extract `userId` and `addonId` from URL pathname.
Validate both as UUID format.

DELETE handler:
- Auth: `validateApiKey` -> `rateLimit` -> `requirePermission(key, 'write')`.
- Call `removeAddonFromUser(userId, addonId)`.
- If error or !data, return `errorResponse('NOT_FOUND', 'Assignment not found', 404)`.
- Audit: `{ category: 'admin', action: 'addon.unassign', target_type: 'user_designation', target_id: addonId, description: 'Removed add-on from user via API', metadata: { user_id: userId } }`.
- Return `successResponse(data)`.
