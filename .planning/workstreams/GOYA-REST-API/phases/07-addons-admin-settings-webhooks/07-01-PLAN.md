Create `lib/api/services/addons.ts` following the exact pattern from `events.ts`:

1. Import `getSupabaseService`, `paginationToRange`, `PaginationParams`.
2. Export `ADDONS_SORT_FIELDS = ['created_at', 'updated_at', 'name', 'category', 'priority', 'price_cents']`.
3. Export `VALID_ADDON_CATEGORIES = ['teacher_designation', 'experienced_teacher', 'school_designation', 'special']` for reuse in route handler validation.

**listAddons(params: ListAddonsParams)** — Per ADON-01
- Interface `ListAddonsParams`: `{ pagination: PaginationParams; category?: string; search?: string }`
- Query `products` table with `select('*', { count: 'exact' })`.
- Filter `.eq('is_active', true)` by default (only return active products in list).
- Optional `.eq('category', category)` filter.
- Optional `.or('name.ilike.%${search}%,full_name.ilike.%${search}%,slug.ilike.%${search}%')` search filter.
- Apply `.order()` and `.range()` from pagination params.
- Return `{ data, count, error }`.

**getAddonById(id: string)** — Per ADON-02
- Query `products` table, `.eq('id', id).single()`.
- Do NOT filter by `is_active` — detail endpoint should return even inactive products (admin may need to view).
- Return `{ data, error }`.

**createAddon(params: CreateAddonParams)** — Per ADON-03
- Interface `CreateAddonParams`: `{ slug: string; name: string; full_name: string; category: string; price_display: string; price_cents?: number; image_path?: string | null; description?: string | null; features?: unknown; requires_any_of?: string[]; hidden_if_has_any?: string[]; has_variants?: boolean; variants?: unknown; priority?: number; is_active?: boolean }`.
- Insert into `products`, `.select().single()`.
- Return `{ data, error }`.

**updateAddon(id: string, updates: UpdateAddonParams)** — Per ADON-04
- Interface `UpdateAddonParams` with all fields optional except id.
- Export `ALLOWED_ADDON_UPDATE_FIELDS` array: `['name', 'full_name', 'category', 'price_display', 'price_cents', 'image_path', 'description', 'features', 'requires_any_of', 'hidden_if_has_any', 'has_variants', 'variants', 'priority', 'is_active']` (NOT slug — slug is immutable).
- Validate only allowed fields (same pattern as events.ts `updateEvent`).
- Update with `.eq('id', id).select().single()`.
- Return `{ data, error }`.

**deleteAddon(id: string)** — Per ADON-05
- Soft-delete by setting `is_active = false` (products table uses `is_active` flag, not `deleted_at`).
- Query: `.update({ is_active: false }).eq('id', id).eq('is_active', true).select().single()`.
- Return `{ data, error }`.

Use `getSupabaseService() as any` pattern consistent with all other services.

**app/api/v1/addons/route.ts** — GET (ADON-01) + POST (ADON-03):

GET handler:
- Auth: `validateApiKey` -> `rateLimit` -> `requirePermission(key, 'read')`.
- Parse pagination with `parsePaginationParams(ctx.url, ADDONS_SORT_FIELDS)`.
- Extract optional filters: `category` (validate against `VALID_ADDON_CATEGORIES`), `search`.
- Call `listAddons({ pagination, category, search })`.
- Return `paginatedResponse(data, meta)`.

POST handler:
- Auth: `validateApiKey` -> `rateLimit` -> `requirePermission(key, 'write')`.
- Parse JSON body with try/catch (400 on invalid JSON).
- Validate required fields: `slug` (non-empty string), `name` (non-empty string), `full_name` (non-empty string), `category` (must be in VALID_ADDON_CATEGORIES), `price_display` (non-empty string).
- Build params object spreading only known fields.
- Call `createAddon(params)`.
- On success: `ctx.logAudit({ category: 'admin', action: 'addon.create', target_type: 'product', target_id: data.id, description: 'Created add-on via API', metadata: { name: data.name } })`.
- Return `successResponse(data, 201)`.

**app/api/v1/addons/[id]/route.ts** — GET (ADON-02) + PATCH (ADON-04) + DELETE (ADON-05):

Extract `id` from URL pathname: `const id = ctx.url.pathname.split('/').at(-1)!;`
Validate UUID format with regex `/^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i` — return 400 if invalid.

GET handler:
- Auth: `validateApiKey` -> `rateLimit` -> `requirePermission(key, 'read')`.
- Call `getAddonById(id)`.
- If error or !data return `errorResponse('NOT_FOUND', 'Add-on not found', 404)`.
- Return `successResponse(data)`.

PATCH handler:
- Auth: `validateApiKey` -> `rateLimit` -> `requirePermission(key, 'write')`.
- Parse JSON body.
- Check unknown fields: compare body keys against `ALLOWED_ADDON_UPDATE_FIELDS`, return 400 on unknowns.
- If `category` provided, validate against `VALID_ADDON_CATEGORIES`.
- Call `updateAddon(id, body)`.
- On error: if `updateAddon` returns error, call `getAddonById(id)` to distinguish 404 from 500.
- Audit log: `{ category: 'admin', action: 'addon.update', target_type: 'product', target_id: data.id, description: 'Updated add-on via API', metadata: { fields: Object.keys(body) } }`.
- Return `successResponse(data)`.

DELETE handler:
- Auth: `validateApiKey` -> `rateLimit` -> `requirePermission(key, 'write')`.
- Call `deleteAddon(id)`.
- If error or !data: call `getAddonById(id)` to check if already deleted vs not found.
- Audit log: `{ category: 'admin', action: 'addon.delete', target_type: 'product', target_id: id, description: 'Soft-deleted add-on via API' }`.
- Return `successResponse(data)`.
