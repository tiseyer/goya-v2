**Interfaces:**
```typescript
export interface WebhookTriggerPayload {
  type: string;        // required — event type identifier (e.g., "user.updated", "course.completed")
  payload: Record<string, unknown>;  // required — arbitrary JSON payload
  timestamp?: string;  // optional ISO timestamp
}

export interface WebhookPaymentPayload {
  transaction_id: string;   // required — external payment reference
  amount_cents: number;     // required — payment amount in cents
  currency: string;         // required — ISO currency code (e.g., "USD")
  user_id?: string;         // optional — GOYA user ID if known
  status: string;           // required — "completed" | "refunded" | "failed"
  metadata?: Record<string, unknown>;
}

export interface WebhookNotifyPayload {
  user_ids: string[];       // required — at least one user ID
  message: string;          // required — notification message
  channel?: string;         // optional — "email" | "in_app" (default: "in_app")
  metadata?: Record<string, unknown>;
}
```

**processWebhookTrigger(payload: WebhookTriggerPayload)** — Per WHKN-01
- Validate: `type` must be non-empty string, `payload` must be an object.
- Log to console: `[webhook:trigger] type=${payload.type}`.
- Return `{ success: true, event_type: payload.type, received_at: new Date().toISOString() }`.

**processWebhookPayment(payload: WebhookPaymentPayload)** — Per WHKN-02
- Validate: `transaction_id` non-empty string, `amount_cents` must be a number, `currency` non-empty string (3 chars), `status` must be one of `['completed', 'refunded', 'failed']`.
- Log to console: `[webhook:payment] txn=${payload.transaction_id} status=${payload.status}`.
- Return `{ success: true, transaction_id: payload.transaction_id, status: payload.status, received_at: new Date().toISOString() }`.

**processWebhookNotify(payload: WebhookNotifyPayload)** — Per WHKN-03
- Validate: `user_ids` must be non-empty array of strings, `message` must be non-empty string.
- If `channel` provided, must be one of `['email', 'in_app']`.
- Log to console: `[webhook:notify] users=${payload.user_ids.length} channel=${payload.channel ?? 'in_app'}`.
- Return `{ success: true, notified_count: payload.user_ids.length, channel: payload.channel ?? 'in_app', received_at: new Date().toISOString() }`.

Each function returns `{ data, error }` where error is a string message if validation fails, null on success.

Create three route files, one per webhook type. All follow the same pattern.

**Common pattern for all three routes:**
- Auth: `validateApiKey` -> `rateLimit` -> `requirePermission(key, 'write')` (webhooks are write operations).
- Parse JSON body with try/catch (return 400 on invalid JSON).
- Call the corresponding service function.
- If service returns an error (validation failure), return `errorResponse('INVALID_PAYLOAD', error, 400)`.
- Audit log on success.
- Return `successResponse(data)`.
