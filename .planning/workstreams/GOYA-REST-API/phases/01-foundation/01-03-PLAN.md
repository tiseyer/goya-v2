  <action>
Create `lib/api/middleware.ts` with these three exported functions. This is NOT Next.js edge middleware — these are composable functions called inside route handlers (via the handler factory or directly).

**1. validateApiKey** — Validates the `x-api-key` header against the api_keys table:
- Extract key from `x-api-key` header (or `Authorization: Bearer ...` as fallback)
- If no key provided: return `errorResponse('UNAUTHORIZED', 'API key required', 401)`
- Hash the key using Node.js `crypto.createHash('sha256').update(key).digest('hex')`
- Query `api_keys` table: `SELECT * FROM api_keys WHERE key_hash = $hash AND active = true`
- If no match: return `errorResponse('UNAUTHORIZED', 'Invalid API key', 401)`
- If match: update `last_used_at` to now() and increment `request_count` (fire-and-forget, no await needed)
- Return the ApiKeyRow on success (not a Response)
- Function signature: `async function validateApiKey(req: NextRequest): Promise<ApiKeyRow | Response>`

**2. rateLimit** — In-memory sliding window rate limiter per API key:
- Use a `Map<string, { count: number; resetAt: number }>` at module scope
- Window: 60 seconds. Limit: 100 requests per key.
- On each call: check if window expired (reset if so), increment count
- If count > 100: return `errorResponse('RATE_LIMITED', 'Rate limit exceeded. Max 100 requests per minute.', 429, { retry_after: secondsUntilReset })`
- Also set `Retry-After` header on the 429 response
- Return null if under limit
- Function signature: `function rateLimit(keyId: string): Response | null`
- Periodically clean stale entries (every 1000 calls, delete entries past their resetAt)

**3. requirePermission** — Checks API key has the required permission:
- Accepts an ApiKeyRow and a required ApiKeyPermission
- If `apiKey.permissions` includes `'admin'`, always allow (admin supersedes all)
- If `apiKey.permissions` includes the required permission, allow
- Otherwise: return `errorResponse('FORBIDDEN', 'Insufficient permissions', 403)`
- Function signature: `function requirePermission(apiKey: ApiKeyRow, required: ApiKeyPermission): Response | null`

**Composition pattern** (document in JSDoc):
```typescript
// In a route handler:
const keyOrError = await validateApiKey(ctx.req);
if (keyOrError instanceof Response) return keyOrError;
const limited = rateLimit(keyOrError.id);
if (limited) return limited;
const forbidden = requirePermission(keyOrError, 'write');
if (forbidden) return forbidden;
// ... proceed with business logic
```
