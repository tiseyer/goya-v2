**Part A: Create migration `supabase/migrations/20260350_courses_soft_delete.sql`:**

```sql
-- ============================================================
-- Migration: Add soft-delete support to courses table
-- ============================================================

-- Add deleted_at column for soft-delete
ALTER TABLE public.courses ADD COLUMN IF NOT EXISTS deleted_at timestamptz DEFAULT null;

-- Expand status CHECK constraint to include 'deleted'
ALTER TABLE public.courses DROP CONSTRAINT IF EXISTS courses_status_check;
ALTER TABLE public.courses ADD CONSTRAINT courses_status_check CHECK (status IN ('published', 'draft', 'deleted'));
```

const VALID_CATEGORIES: CourseCategory[] = ['Workshop', 'Yoga Sequence', 'Dharma Talk', 'Music Playlist', 'Research'];
const VALID_LEVELS: CourseLevel[] = ['Beginner', 'Intermediate', 'Advanced', 'All Levels'];

**GET handler** (CRSE-01):
1. Auth + rate limit + requirePermission('read') -- same 3-step pattern as events
2. Parse pagination: `parsePaginationParams(ctx.url, COURSES_SORT_FIELDS)`
3. Extract filter params from ctx.url.searchParams:
   - `category` -- validate against VALID_CATEGORIES, ignore if invalid
   - `level` -- validate against VALID_LEVELS, ignore if invalid
   - `access` -- validate against VALID_ACCESS, ignore if invalid
   - `status` -- validate against VALID_STATUSES, ignore if invalid
   - `search` -- pass through as string
4. Call `listCourses({ pagination, category, level, access, status, search })`
5. If error: `errorResponse('QUERY_ERROR', 'Failed to fetch courses', 500)`
6. Build pagination meta, return `paginatedResponse(data, meta)`

**POST handler** (CRSE-03):
1. Auth + rate limit + requirePermission('write')
2. Parse JSON body (try/catch, 400 on failure)
3. Validate required fields: `title` (non-empty string), `category` (in VALID_CATEGORIES). Return 400 with specific error message for each missing/invalid required field.
4. Validate optional enum fields if present:
   - `level` against VALID_LEVELS
   - `access` against VALID_ACCESS
   - `status` against VALID_STATUSES
5. Build CreateCourseParams from validated body -- only pass through known fields (title, category, short_description, description, instructor, duration, level, access, vimeo_url, thumbnail_url, gradient_from, gradient_to, status)
6. Call `createCourse(params)`
7. If error: `errorResponse('CREATE_ERROR', 'Failed to create course', 500)`
8. Audit log: `await ctx.logAudit({ category: 'admin', action: 'course.create', target_type: 'course', target_id: data.id, description: 'Created course via API', metadata: { title: data.title } })`
9. Return `successResponse(data, 201)`

Export: `export const GET = handlers.GET; export const POST = handlers.POST;`
