**GET handler** (CRSE-06):

3. Validate courseId UUID format (return 400 if invalid)

5. Call `listEnrollments(courseId, pagination)`

   - If error === 'COURSE_NOT_FOUND': `errorResponse('NOT_FOUND', 'Course not found', 404)`
   - Other errors: `errorResponse('QUERY_ERROR', 'Failed to fetch enrollments', 500)`
7. Build pagination meta, return `paginatedResponse(data, meta)`

**POST handler** (CRSE-07):
1. Auth + rate limit + requirePermission('write')
2. Extract courseId from URL (same as GET)
3. Validate courseId UUID format (return 400 if invalid)
4. Parse JSON body -- expect `{ user_id: string }`
5. Validate user_id is present and is a valid UUID format. Return 400 if missing or invalid.
6. Call `enrollUser(courseId, userId)`
7. Handle service errors:
   - If error === 'COURSE_NOT_FOUND': `errorResponse('NOT_FOUND', 'Course not found', 404)`
   - If error === 'ALREADY_ENROLLED': `errorResponse('CONFLICT', 'User is already enrolled in this course', 409)`
   - Other errors: `errorResponse('ENROLLMENT_ERROR', 'Failed to enroll user', 500)`
8. Audit log: `await ctx.logAudit({ category: 'admin', action: 'course.enroll', target_type: 'enrollment', target_id: courseId, description: 'Enrolled user in course via API', metadata: { user_id: userId, course_id: courseId } })`
