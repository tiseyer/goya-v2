**`listUsers(params)`** — per USER-01:
- Accept an object: `{ pagination: PaginationParams; role?: UserRole; subscription_status?: SubscriptionStatus; search?: string; date_from?: string; date_to?: string }`
- Use `getSupabaseService()` with `as any` cast (same pattern as middleware.ts — profiles not in generated types)
- Build query: `supabase.from('profiles').select('*', { count: 'exact' })`
- Apply filters conditionally:
  - `.eq('role', role)` if role provided
  - `.eq('subscription_status', subscription_status)` if provided
  - `.gte('created_at', date_from)` if provided
  - `.lte('created_at', date_to)` if provided
  - `.or(`full_name.ilike.%${search}%,email.ilike.%${search}%,username.ilike.%${search}%`)` if search provided
- Apply sort: `.order(pagination.sort, { ascending: pagination.order === 'asc' })`
- Apply range: use `paginationToRange(pagination)` for `.range(from, to)`
- Return `{ data, count, error }` — let route handler build the response

**`getUserById(id: string)`** — per USER-02:
- Query: `supabase.from('profiles').select('*').eq('id', id).single()`
- Return `{ data, error }` — let route handler handle 404 if data is null

Allowed sort fields for list: `['created_at', 'updated_at', 'full_name', 'email', 'role']`

Export `USERS_SORT_FIELDS` array constant for use by the route handler.

**app/api/v1/users/route.ts** — per USER-01:
```
import { createApiHandler } from '@/lib/api/handler';
import { validateApiKey, rateLimit, requirePermission } from '@/lib/api/middleware';
import { paginatedResponse, errorResponse } from '@/lib/api/response';
import { parsePaginationParams, buildPaginationMeta } from '@/lib/api/pagination';
import { listUsers, USERS_SORT_FIELDS } from '@/lib/api/services/users';
import type { UserRole, SubscriptionStatus } from '@/lib/types';
```

GET handler:
1. Validate API key: `const keyOrError = await validateApiKey(ctx.req); if (keyOrError instanceof Response) return keyOrError;`
2. Rate limit: `const limited = rateLimit(keyOrError.id); if (limited) return limited;`
3. Permission check: `const forbidden = requirePermission(keyOrError, 'read'); if (forbidden) return forbidden;`
4. Parse pagination: `parsePaginationParams(ctx.url, USERS_SORT_FIELDS)`
5. Extract filter params from ctx.url.searchParams: `role` (validate against UserRole values), `status` (subscription_status), `search`, `date_from`, `date_to`
6. Call `listUsers({ pagination, role, subscription_status, search, date_from, date_to })`
7. If error, return `errorResponse('QUERY_ERROR', 'Failed to fetch users', 500)`
8. Build pagination meta with `buildPaginationMeta(count ?? 0, pagination)`
9. Return `paginatedResponse(data, meta)`

Export: `export const GET = handlers.GET;`

- Create directory `app/api/v1/users/[id]/`

- GET handler:

  7. Return `successResponse(data)`
