| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Button label | 14px (text-sm) | 600 (font-semibold) | n/a (single line) |
| Helper text | 12px (text-xs) | 400 (normal) | 1.5 |
| Card meta label | 10px (text-[10px]) | 600 (font-semibold, uppercase, tracking-widest) | n/a |
| Section heading | 16px (text-base) | 600 (font-semibold) | 1.2 |

Rules:
- Button labels use font-semibold (600) for all variants.
- Helper text (e.g. "{firstName} wants to connect") uses text-xs at weight 400, color slate-400 / text-slate-500.
- No Display size needed in this phase — buttons only, no new headings.

Accent reserved for: primary CTA button (`bg-primary-light` = `#4e87a0`) only. Hover darkens to `bg-primary` (#345c83). Active state darkens to `bg-primary-dark` (#1e3a52).

The existing member card uses `bg-[#1B3A5C]` (primary-dark) as its surface — buttons rendered inside it must read clearly against this dark background. Primary button uses white text on `bg-primary-light`.

This phase adds role-awareness to ConnectButton. The following states are the full contract:

### Viewer × Profile role combinations

| Viewer Role | Profile Role | Button Label | Button Variant | Connection Type Sent |
|-------------|--------------|--------------|----------------|----------------------|
| student | teacher | "Request Mentorship" | primary (bg-primary-light) | `mentorship` |
| teacher | school | "Apply as Faculty" | primary (bg-primary-light) | `faculty` |
| wellness_practitioner | school | "Apply as Faculty" | primary (bg-primary-light) | `faculty` |
| teacher | teacher | "Connect with {firstName}" | primary (bg-primary-light) | `peer` |

| Status | Display | Style |

| accepted | "Connected" + checkmark + "Message" below | Connected: emerald-50 bg / emerald-700 text / emerald-200 border, disabled; Message: secondary |

## Interaction Contract

### Loading / async states

- When `sendRequest` is in-flight: show Button with `loading={true}` prop (spins built-in spinner). Disable further clicks.
- When `acceptRequest` or `declineRequest` is in-flight: show loading spinner on the tapped button only; leave the other button enabled.
- No skeleton loader needed — ConnectButton renders after parent page data is available.

### "Manage School" navigation

- Renders as a secondary `<Button>` (border variant) with an `onClick` that pushes router to `/settings`.
- Does NOT call `sendRequest`. Does not alter connection state.

### Hiding button for own profile

- If `viewerRole` and `profileOwnerId` match the logged-in user, render `null` from ConnectButton. No placeholder div.

### Accessibility

- All interactive buttons must have accessible text. Icon-only states (clock, check) must have `aria-hidden="true"` on the SVG and a visually hidden text label via `sr-only` span.
- Disabled buttons retain `aria-disabled="true"` and `disabled` attribute.
- Focus ring: `focus-visible:ring-2 focus-visible:ring-primary/40` — already built into Button component.
