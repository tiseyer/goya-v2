  <action>
    Modify `app/members/[id]/page.tsx` to add three things: (A) viewer profile fetch, (B) school ownership check, (C) new props on ConnectButton.

    **A. Fetch the viewer's profile to get their member_type (after line 33, after `currentUserId` is set):**

    ```typescript
    // Fetch viewer's profile for role-aware connect button
    const { data: viewerProfile } = currentUserId
      ? await supabase
          .from('profiles')
          .select('member_type, role')
          .eq('id', currentUserId)
          .single()
      : { data: null };

    const viewerRole = viewerProfile
      ? (viewerProfile.member_type ?? viewerProfile.role ?? 'student')
      : null;
    ```

    This uses the same fallback chain as the profile's own role derivation (line 90): `member_type ?? role ?? 'student'`. When `currentUserId` is null (unauthenticated), `viewerRole` is null.

    **B. Check school ownership (after the viewerRole computation, before the return statement):**

    Add this after the `role` variable is computed (after line 90):

    ```typescript
    // School ownership check — only query when viewing a school profile while authenticated
    let isOwnSchool = false;
    if (role === 'school' && currentUserId) {
      const { data: ownedSchool } = await supabase
        .from('schools')
        .select('id')
        .eq('owner_id', currentUserId)
        .maybeSingle();
      isOwnSchool = !!ownedSchool;
    }
    ```

    NOTE on schema: The `schools` table has no `profile_id` column linking back to `profiles`. A school profile in the member directory comes from either (a) static data with `role: 'School'` mapped to `'school'`, or (b) a real profile row whose `role` or `member_type` indicates school. Since `member_type` doesn't include `'school'`, school profiles are identified by the `STATIC_ROLE_MAP` or by `profile.role`. The ownership check queries whether the current viewer owns ANY school via `schools.owner_id`. For v1.1 this is sufficient because a teacher typically owns one school. If viewing a school profile and the viewer owns a school, they see "Manage School". No `profile.id` to `schools.id` join is needed — the `owner_id` check is enough.
