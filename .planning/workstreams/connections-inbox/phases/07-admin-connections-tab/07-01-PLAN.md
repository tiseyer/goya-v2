    **File 1: `app/actions/adminConnections.ts`**

    Create a server action file with `'use server'` directive at top.

    Export `removeConnectionAsAdmin(connectionId: string, userId: string)`:
    1. Import `createSupabaseServerClient` from `@/lib/supabaseServer` and `getSupabaseService` from `@/lib/supabase/service`
    2. Import `redirect` from `next/navigation` and `revalidatePath` from `next/cache`
    3. Create session client via `await createSupabaseServerClient()`
    4. Get current user via `supabase.auth.getUser()` — if no user, `redirect('/sign-in')`
    5. Fetch caller's profile role: `supabase.from('profiles').select('role').eq('id', user.id).single()`
    6. Guard: if role is NOT `'admin'` and NOT `'moderator'`, `redirect('/')`
    7. Delete the connection: `await (getSupabaseService() as any).from('connections').delete().eq('id', connectionId)`
    8. Call `revalidatePath('/admin/users/' + userId, 'page')` to refresh the server component cache
    9. Return nothing (void)

    The `userId` parameter is for the revalidatePath call so the page refreshes correctly. The `as any` cast on getSupabaseService is needed because the service client type does not know the DB schema.

    **File 2: `app/admin/users/[id]/RemoveConnectionButton.tsx`**

    Create a `'use client'` component mirroring ResetOnboardingButton's confirm pattern.

    Props: `{ connectionId: string; userId: string }`

    State: `confirming` (boolean, default false), `loading` (boolean, default false)

    Import `removeConnectionAsAdmin` from `@/app/actions/adminConnections`, `Button` from `@/app/components/ui/Button`, and `useState` from `react`.

    Render logic:
    - Default state: A single `<Button variant="ghost" size="sm">` showing "Remove"
    - On click: set `confirming = true`
    - Confirming state: Show a `<div className="flex items-center gap-2">` with:
      - `<Button variant="danger" size="sm" loading={loading}>` showing "Confirm" — on click: set loading=true, call `await removeConnectionAsAdmin(connectionId, userId)`, set loading=false and confirming=false
      - `<Button variant="ghost" size="sm">` showing "Cancel" — on click: set confirming=false

    Do NOT use `router.refresh()` here — the server action already calls `revalidatePath` which handles cache invalidation.

    **3. Fetch connections when tab is 'connections'**:
    After the existing profile fetch and before the return statement, add conditionally:
    ```typescript
    let connections: any[] = [];
    if (tab === 'connections') {
      const { data } = await (getSupabaseService() as any)
        .from('connections')
        .select(`
          id, type, status, created_at,
          requester:profiles!connections_requester_id_fkey(id, full_name, avatar_url),
          recipient:profiles!connections_recipient_id_fkey(id, full_name, avatar_url)
        `)
    ```

    ```tsx
            return (
              <div
                key={conn.id}
                className="flex items-center justify-between px-4 py-3 border-b border-[#F3F4F6] last:border-b-0"
              >

                    />
                  ) : (
                    <div className="w-10 h-10 rounded-full bg-slate-100 flex items-center justify-center text-sm font-bold text-slate-400">

                  <div>
                    <p className="text-sm font-medium text-[#1B3A5C]">

                      <Badge variant={conn.status === 'accepted' ? 'subtle' : 'muted'} size="sm">

                      </Badge>
    ```
