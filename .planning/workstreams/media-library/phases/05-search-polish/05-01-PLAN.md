### Task 3: POLISH-03 ‚Äî Smooth Detail Panel Animations
**Type:** auto

Add CSS slide-in/out transitions to:
1. `MediaDetailPanel` ‚Äî the `w-[380px]` div currently has no transition. Wrap open/close in a mount animation using CSS `translate-x` with `transition-transform`.
2. `MemberMediaDetailPanel` ‚Äî same treatment on the `w-[340px]` div.

**Approach:** Since both panels are conditionally rendered with `{selectedItem && <Panel />}`, we cannot animate unmounting with pure CSS. Instead, use an `isVisible` state that lags behind `selectedItem` by a frame ‚Äî render the panel when `selectedItem !== null`, animate it in immediately, and animate it out before removing from DOM.

Implementation: In `MediaPageClient` and `MemberMediaClient`, lift the animation into the parent using `useRef`/state to track "panel is closing" vs "panel is open". The panel itself gets a `data-closing` prop or a CSS class that triggers the slide-out animation.

Simpler approach: Use Tailwind `translate-x-full` ‚Üí `translate-x-0` with `transition-transform duration-200`. The panel always renders when `selectedItem` is set, so slide-in works immediately. For slide-out, we keep the last selected item in a `useRef` and render the panel with `selectedItem || lastItemRef.current` while using a boolean to control the CSS class.

**Files modified:**
- `app/admin/media/MediaDetailPanel.tsx` ‚Äî add `isClosing?: boolean` prop, apply `translate-x-full` when closing
- `app/admin/media/MediaPageClient.tsx` ‚Äî manage `panelClosing` state, delay `setSelectedItem(null)` by 200ms
- `app/settings/media/MemberMediaDetailPanel.tsx` ‚Äî same `isClosing` prop
- `app/settings/media/MemberMediaClient.tsx` ‚Äî manage panel closing state

**Done:** Both panels animate smoothly in (200ms) and out (200ms) on open/close.
