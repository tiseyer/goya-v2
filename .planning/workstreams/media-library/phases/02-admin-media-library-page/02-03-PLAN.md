  <name>Task 1: Server actions for update, delete, and upload</name>
  <files>app/admin/media/actions.ts</files>
  <action>
    Add three server actions to `app/admin/media/actions.ts` (alongside existing getFolders and getMediaItems):

    **updateMediaItem(id, updates):**
    - Use `getSupabaseService()` to update `media_items` where `id = id`
    - Only update fields present in updates (title, alt_text, caption)
    - Also set `updated_at = new Date().toISOString()`
    - Return `{ success: true }` or `{ success: false, error: error.message }`

    **deleteMediaItem(id, filePath):**
    - Use `getSupabaseService()` to:
      1. First delete from Supabase Storage: `supabase.storage.from(bucket).remove([filePath])`
         — need to derive bucket from the media_items row. Fetch the row first to get bucket + file_path.
      2. Then delete from media_items: `.from('media_items').delete().eq('id', id)`
    - Return `{ success: true }` or `{ success: false, error: error.message }`

    **uploadMediaItem(params):**
    - Determine storage path: `${params.bucket}/${Date.now()}-${params.file.name.replace(/\s+/g, '-')}`
    - Upload to Supabase Storage using `getSupabaseService().storage.from(params.bucket).upload(path, params.file, { contentType: params.file.type, upsert: false })`
    - On upload success, get public URL: `supabase.storage.from(params.bucket).getPublicUrl(path).data.publicUrl`
    - Insert into media_items with all fields including: bucket, file_name, file_path (= storage path), file_url (= public URL), file_type (= params.file.type), file_size (= params.file.size), width, height, uploaded_by (get from supabase auth in server context or pass as param), uploaded_by_role, folder (= params.folder ?? null)
    - Check if `registerMediaItemAction` exists in the codebase from Phase 1 (search lib/ or app/admin/media/). If it does, call it instead of directly inserting. If it doesn't exist yet, do the direct insert.
    - Return `{ success: true, item: insertedRow }` or `{ success: false, error: message }`

    Note: Server actions cannot stream upload progress to the client. Progress will be simulated client-side (0% → uploading → 100% on promise resolve). This is acceptable for the initial implementation.
  </action>

    This component has two responsibilities:
    1. Render an Upload button (passed as trigger prop or rendered inline)
    2. Manage drag-drop state

    Props: `{ activeFolder, activeBucket, onUploadComplete, onUploadProgress, onUploadStart, children }` where children is the grid/list area to wrap with drag detection.

    Drag-drop:
    - Wrap children in a `<div>` with `onDragOver`, `onDragEnter`, `onDragLeave`, `onDrop` handlers
    - `isDragOver` state: true while dragging files over the area
    - When isDragOver: show overlay `<div className="absolute inset-0 bg-primary/10 border-2 border-dashed border-primary rounded-lg z-20 flex items-center justify-center pointer-events-none"><p>Drop files to upload</p></div>` (parent must have relative positioning)

    - Upload button: `<button onClick={() => fileInputRef.current?.click()} className="..."><UploadIcon /> Upload</button>` — render this separately outside the drag zone (in toolbar)

    `handleFiles(files: FileList)`:
    - Filter by type (accepted MIME types) and size (<= 50MB = 50 * 1024 * 1024 bytes). Log skipped files to console.
    - For each valid file:
      1. Generate `fileId = crypto.randomUUID()`
      2. Call `onUploadStart(fileId, file.name)`
      3. If image (file.type.startsWith('image/')):
         - Extract dimensions client-side: `new Promise((resolve) => { const img = new Image(); img.onload = () => resolve({ width: img.naturalWidth, height: img.naturalHeight }); img.src = URL.createObjectURL(file); })`
      4. Call `onUploadProgress(fileId, 30)` to show activity
      5. Await `uploadMediaItem({ file, bucket: activeBucket, folder: activeFolder, width, height, uploadedByRole })` — note uploadedByRole needs to come from session. Import `createClient` from supabase client-side utils and call `supabase.auth.getUser()` to get role from user metadata, or accept it as a prop from MediaPageClient which has session data.
      6. Call `onUploadProgress(fileId, 100)`
      7. On success: call `onUploadComplete(result.item!)`
      8. Process files sequentially (await each before next) to avoid overwhelming storage

    **Update `app/admin/media/MediaPageClient.tsx`**:

    Add `uploadCards: UploadCard[]` state.
