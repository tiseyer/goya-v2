# Phase 4, Plan 1: Folder Management

## Objective

Admins and moderators can organise media into a custom folder hierarchy within each bucket.
Specifically: create folder via modal, rename inline on double-click, reorder via drag-and-drop,
and delete folder (admin only) with confirmation and file-count warning.

## Context

- FolderSidebar.tsx: existing sidebar component rendering folders per bucket
- actions.ts: server actions; getFolders() exists; needs createFolder, updateFolder, deleteFolder, reorderFolders
- media_folders table has: id, name, parent_id, bucket, sort_order, created_by, created_at
- @dnd-kit/core + @dnd-kit/sortable are installed

## Tasks

### Task 1 (type="auto"): Add folder CRUD server actions

Add four server actions to `app/admin/media/actions.ts`:
- `createFolder(name, bucket, parentId?, createdBy)` — inserts a row, assigns sort_order = max+1
- `updateFolder(id, name)` — renames a folder
- `deleteFolder(id)` — validates no files exist in this folder first (returns fileCount warning), then deletes
- `reorderFolders(ids)` — updates sort_order for an array of folder IDs in order

Done criteria: TypeScript compiles without errors on the four new exports.

### Task 2 (type="auto"): Create folder modal component

Create `app/admin/media/components/CreateFolderModal.tsx`:
- Modal overlay with name input + parent selector (dropdown of existing folders in the same bucket)
- Cancel / Create buttons
- Calls createFolder action on submit, then calls onCreated callback
- Shows inline validation error if name is empty

Done criteria: Component renders without TypeScript errors, integrates with createFolder action.

### Task 3 (type="auto"): Extend FolderSidebar with management controls

Rewrite `app/admin/media/FolderSidebar.tsx` to add all management interactions:
- (+) button in sidebar header (next to "Folders" label, visible when not collapsed) — opens CreateFolderModal
- Double-click on custom folder name → inline text input for rename (calls updateFolder on blur/Enter)
- Drag-and-drop reorder of folders within the same bucket using @dnd-kit/sortable (calls reorderFolders)
- Delete button (trash icon) on hover for admin users (admin-only): shows confirmation dialog with file count warning, calls deleteFolder

Props additions: `isAdmin: boolean`, `currentUserId: string`, `onFoldersChange: (folders: MediaFolder[]) => void`

Done criteria: Component renders, all interactions are wired to server actions, onFoldersChange is called after mutations.

### Task 4 (type="auto"): Wire MediaPageClient to new FolderSidebar props

Update `app/admin/media/MediaPageClient.tsx`:
- Add `folders` state (starts from `initialFolders`, updated via `onFoldersChange`)
- Pass `isAdmin`, `currentUserId`, `onFoldersChange` to FolderSidebar
- After folder mutations, the local folders state is updated so sidebar refreshes without full page reload

Done criteria: `npx tsc --noEmit` passes. Folder creation/rename/reorder/delete all update the UI immediately.

## Verification
