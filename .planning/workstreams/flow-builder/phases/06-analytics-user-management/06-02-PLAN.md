  <action>
    **GET /api/admin/flows/user-flows** (`route.ts`):
    Accept query param `userId` (required). Use `requireFlowAdmin()` for auth. Return 400 if userId missing.
    Query `flow_responses` joined with `flows` (flow name, status) for the given user_id. Use service role client (`getSupabaseService`) to bypass RLS since admin needs to read any user's responses.
    Return JSON array:
    ```typescript
    Array<{
      id: string;          // flow_response id
      flowId: string;
      flowName: string;
      flowStatus: FlowStatus;
      status: FlowResponseStatus;
      startedAt: string | null;
      completedAt: string | null;
    }>
    ```
    Order by started_at descending (most recent first).

    **DELETE /api/admin/flows/user-flows/[responseId]** (`[responseId]/route.ts`):
    Use `requireFlowAdmin()`. Extract responseId from URL params.
    Delete the flow_response row with this id using service role client. This "resets" the user's flow so it will be shown again on next trigger evaluation (per USERMGMT-02).
    Also delete related rows from `flow_action_executions` for the same flow_id + user_id (so actions re-fire on re-display).
    Return 200 `{ success: true }` or 404 if not found.

    **PATCH /api/admin/flows/user-flows/[responseId]** (same file):
    Use `requireFlowAdmin()`. Accept body `{ status: 'completed' }`.
    Update flow_response: set status='completed', completed_at=now(). Use service role client.
    Return 200 with updated response or 404 if not found (per USERMGMT-03 mark-complete).

    **POST /api/admin/flows/user-flows/assign** (`assign/route.ts`):
    Use `requireFlowAdmin()`. Accept body `{ userId: string, flowId: string, markComplete?: boolean }`.
    Validate both IDs exist. If markComplete is true, insert a flow_response with status='completed', started_at=now(), completed_at=now(). Otherwise insert with status='in_progress', started_at=now() (per USERMGMT-03 force-assign).
    Use upsert on (flow_id, user_id) unique constraint — if response already exists, update it.
    Return 201 with the created/updated response.
  </action>
