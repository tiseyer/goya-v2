    **API route** (`app/api/admin/flows/[id]/analytics/route.ts`):
    Create a GET endpoint that accepts query params `from` and `to` (ISO date strings). Use `requireFlowAdmin()` for auth (same pattern as stats/route.ts). Query `flow_analytics` table filtered by flow_id from URL param and created_at between from/to. Also query `flow_steps` for this flow (id, position, title) to label the drop-off chart.

    Return JSON shape:
    ```typescript
    {
      counts: { shown: number, started: number, completed: number, skipped: number, dismissed: number },
      completionRate: number, // completed / shown, 0 if shown=0
      dropoff: Array<{ stepId: string, position: number, title: string | null, reached: number }>
    }
    ```

    For `counts`: aggregate by event type using a single query, count rows per event type.
    For `dropoff`: count distinct user_id per step_id from `step_completed` events, join with flow_steps to get position/title, order by position ascending. Include step 1 count from `started` events (users who started = reached step 1).
    Use `createSupabaseServerClient` (not service role) since admin RLS allows SELECT on flow_analytics.

    **FlowAnalyticsFilters** (`app/admin/flows/components/editor/FlowAnalyticsFilters.tsx`):
    'use client' component. Props: `{ range: string, dateFrom: string, dateTo: string, onChange: (range: string, from: string, to: string) => void }`.
    Render preset buttons as pill-style toggles (not a <select>): Today, Yesterday, This Week, This Month, This Year, Custom.
    When "Custom" is selected, show two date inputs inline.
    On change, call onChange with the new range key and computed ISO from/to dates.
    Use date-fns (already installed) for date math: startOfDay, startOfWeek, startOfMonth, startOfYear, subDays.
    Style: match the select/input styling from AnalyticsFilters.tsx but use pill buttons (`px-3 py-1 rounded-full text-xs font-medium` with active state `bg-[#1B3A5C] text-white` and inactive `bg-slate-100 text-slate-600 hover:bg-slate-200`).

    **FlowAnalyticsDropoff** (`app/admin/flows/components/editor/FlowAnalyticsDropoff.tsx`):
    'use client' component using Recharts BarChart (not LineChart). Props: `{ data: Array<{ stepId: string, position: number, title: string | null, reached: number }> }`.
    X-axis: step labels (position + title or "Step N"). Y-axis: user count. Single bar series for "reached" in color `#1B3A5C`.
    Use ResponsiveContainer, BarChart, Bar, XAxis, YAxis, CartesianGrid, Tooltip from recharts.
    Match the chart container pattern: `w-full h-72 bg-white rounded-lg border p-4`.
    Show "No step data" empty state if data is empty.

    **FlowAnalyticsTab** (`app/admin/flows/components/editor/FlowAnalyticsTab.tsx`):
    'use client' component. Props: `{ flowId: string }`.
    Local state: range (default 'this_month'), dateFrom, dateTo, analyticsData, loading.
    On mount and when date range changes, fetch `/api/admin/flows/${flowId}/analytics?from=${from}&to=${to}`.
    Layout:
    1. FlowAnalyticsFilters at top
    2. Metric cards row (grid 3-col on md, 2-col on sm): Shown, Started, Completed, Skipped, Dismissed, Completion Rate (as percentage with 1 decimal). Use simple card divs styled like AnalyticsMetricCard pattern: `bg-white rounded-2xl border border-slate-100 shadow-sm p-4`.
    3. FlowAnalyticsDropoff chart below metrics with heading "Step Drop-off"
    Show a loading skeleton (pulsing gray rectangles) while fetching.
