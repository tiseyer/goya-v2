There is a bug: the ConditionsBuilder UI allows selecting role values 'student', 'teacher', 'wellness_practitioner', but the condition evaluator only checks `profile.role` which is 'admin'/'moderator'/'member'. The `member_type` field stores the actual student/teacher/wellness_practitioner value.

Fix:

1. In `lib/flows/condition-evaluator.ts`:
   - Add `member_type: string | null` to the `UserProfileForConditions` interface.
   - Update the `role` case in `evaluateSingleCondition`:
     - For `operator === 'equals'`: if value is 'student'|'teacher'|'wellness_practitioner', compare against `userProfile.member_type`; otherwise compare against `userProfile.role`.
     - For `operator === 'in'`: check if userProfile.role OR userProfile.member_type is in the values array.

   Concrete implementation for the `role` case:
   ```typescript
   case 'role': {
     const MEMBER_TYPES = ['student', 'teacher', 'wellness_practitioner'];
     if (condition.operator === 'equals') {
       const v = condition.value as string;
       if (MEMBER_TYPES.includes(v)) {
         return userProfile.member_type === v;
       }
       return userProfile.role === v;
     }
     if (condition.operator === 'in') {
       const values = condition.value as string[];
       return values.includes(userProfile.role) || (userProfile.member_type !== null && values.includes(userProfile.member_type));
     }
     return false;
   }
   ```

2. In `lib/flows/engine.ts`:
   - Add `member_type` to the profile SELECT query on line ~45:
     `.select('role, onboarding_complete, avatar_url, subscription_status, birthday, member_type')`
   - Add `member_type: profile.member_type ?? null` to the `userProfileForConditions` object on line ~76.
