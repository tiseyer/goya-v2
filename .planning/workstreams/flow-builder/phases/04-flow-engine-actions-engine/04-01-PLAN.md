**1. Create migration `supabase/migrations/20260368_flow_action_executions.sql`:**

```sql
CREATE TABLE IF NOT EXISTS public.flow_action_executions (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  flow_id uuid NOT NULL REFERENCES public.flows(id) ON DELETE CASCADE,
  user_id uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  step_id uuid NOT NULL REFERENCES public.flow_steps(id) ON DELETE CASCADE,
  action_type text NOT NULL,
  executed_at timestamptz DEFAULT now() NOT NULL
);

-- Idempotency constraint: one execution per (flow, user, step, action_type)
ALTER TABLE public.flow_action_executions
  ADD CONSTRAINT flow_action_executions_unique
  UNIQUE (flow_id, user_id, step_id, action_type);

-- Index for quick lookups when checking idempotency
CREATE INDEX flow_action_executions_lookup
  ON public.flow_action_executions (flow_id, user_id, step_id);

-- RLS: users can only see their own executions, service role bypasses
ALTER TABLE public.flow_action_executions ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can view own action executions"
  ON public.flow_action_executions FOR SELECT
  USING (auth.uid() = user_id);

CREATE POLICY "Service role manages action executions"
  ON public.flow_action_executions FOR ALL
  USING (auth.role() = 'service_role');
```

**3. Create `lib/flows/condition-evaluator.ts`:**

`import 'server-only'` at top. Export a single function:

```typescript
export function evaluateConditions(
  conditions: FlowCondition[],
  userProfile: UserProfileForConditions
): boolean
```

Define `UserProfileForConditions` interface locally (not exported to types.ts since it is engine-internal):
```typescript
interface UserProfileForConditions {
  role: string;
  onboarding_complete: boolean;
  avatar_url: string | null;
  subscription_status: string | null;
  birthday: string | null; // ISO date
  completed_flow_ids: string[];
}
```

Condition evaluation rules (all conditions are AND-ed — every condition must pass):
- `role`: operator `equals` — check `userProfile.role === value` or `in` — check `(value as string[]).includes(userProfile.role)`
- `onboarding_status`: operator `equals` — check `userProfile.onboarding_complete === (value === 'complete')`
- `has_profile_picture`: operator `equals` — check `(!!userProfile.avatar_url) === value`
- `subscription_status`: operator `equals` — check `userProfile.subscription_status === value` or `in` — check `(value as string[]).includes(userProfile.subscription_status ?? '')`
- `birthday`: operator `is_today` — compare month/day of `userProfile.birthday` to today's month/day (UTC)
- `flow_completed`: operator `completed` — check `userProfile.completed_flow_ids.includes(value as string)`, operator `not_completed` — check `!userProfile.completed_flow_ids.includes(value as string)`

If `conditions` array is empty, return `true` (no conditions = always match).
Unknown condition types return `false` (fail-safe).

**4. Create `lib/flows/engine.ts`:**

`import 'server-only'` at top. Uses `getSupabaseService()` (same pattern as flow-service.ts).

Export three functions:

**`getActiveFlowForUser(userId: string, trigger: FlowTriggerType)`:**
1. Fetch all active flows: `supabase.from('flows').select('*').eq('status', 'active').eq('trigger_type', trigger).order('priority', { ascending: false })`
2. Fetch user profile: `supabase.from('profiles').select('role, onboarding_complete, avatar_url, subscription_status, birthday').eq('id', userId).single()`
3. Fetch user's completed flow IDs: `supabase.from('flow_responses').select('flow_id').eq('user_id', userId).eq('status', 'completed')` — map to string array
4. Fetch user's existing flow_responses (to check frequency): `supabase.from('flow_responses').select('flow_id, status').eq('user_id', userId)`
5. For each flow (already ordered by priority desc), evaluate conditions via `evaluateConditions()`. Also check frequency:
   - `once`: skip if user has ANY flow_response for this flow_id (any status)
   - `every_login` / `every_session`: skip if user has an `in_progress` response (they should complete it first)
   - `custom`: treat as `once` for v1
6. First flow that passes all checks wins. Fetch its steps via `getFlowWithSteps()`.
7. Also fetch/create the user's flow_response record: upsert into `flow_responses` with `status: 'in_progress'`, `started_at: new Date().toISOString()` if none exists.
8. Return `ActiveFlowResponse` (flow WITHOUT conditions field, steps with branches, response record). Strip `conditions` from flow object before returning.

**`recordStepResponse(userId: string, flowId: string, input: StepResponseInput)`:**
1. Fetch the existing flow_response for this user+flow. If none exists, return error.
2. Merge `input.answers` into the existing `responses` JSONB: `{ ...existing.responses, ...input.answers }`
3. Update `flow_responses` with merged responses and `last_step_id: input.step_id`.
4. Return the updated flow_response.

**`completeFlow(userId: string, flowId: string)`:**
1. Update flow_responses set `status: 'completed'`, `completed_at: new Date().toISOString()` where `flow_id = flowId AND user_id = userId`.
2. Insert analytics event `{ flow_id: flowId, user_id: userId, event: 'completed' }` into flow_analytics.
3. Return `{ success: true }` or `{ success: false, error }`.

Create three user-facing API route handlers. These do NOT use requireFlowAdmin — they use the regular user auth pattern: `createSupabaseServerClient()` then `supabase.auth.getUser()`. Unauthenticated = 401.

**`app/api/flows/active/route.ts` — GET handler:**
1. Auth: `const supabase = await createSupabaseServerClient(); const { data: { user } } = await supabase.auth.getUser();` — 401 if no user.
2. Read optional `trigger` query param from request URL, default to `'login'`.
3. Call `getActiveFlowForUser(user.id, trigger)`.
4. If no matching flow, return `NextResponse.json({ flow: null }, { status: 200 })`.
5. If match found, return `NextResponse.json(result, { status: 200 })`.
6. CRITICAL: Verify the returned JSON does NOT include a `conditions` field on the flow object (the engine already strips it, but double-check).

**`app/api/flows/[id]/respond/route.ts` — POST handler:**
1. Auth same pattern — 401 if no user.
2. Extract `id` from `params` (Next.js 15+ async params: `const { id } = await params`).
3. Parse JSON body: `{ step_id: string, answers: Record<string, unknown> }`. Return 400 if missing step_id.
4. Call `recordStepResponse(user.id, id, { step_id, answers })`.
5. Return 200 with updated response, or 404/500 on error.

**`app/api/flows/[id]/complete/route.ts` — POST handler:**
1. Auth same pattern — 401 if no user.
2. Extract `id` from async params.
3. Call `completeFlow(user.id, id)`.
4. Return 200 `{ success: true }` or error response.

All three routes: `import { createSupabaseServerClient } from '@/lib/supabaseServer'`, `import { NextResponse } from 'next/server'`, and relevant engine imports from `@/lib/flows/engine`.
