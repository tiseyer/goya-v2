**Create `app/api/chatbot/generate-answer/route.ts`:**

This is an admin-only streaming endpoint that generates a FAQ answer for a given question.

1. Export `runtime = 'nodejs'` and `maxDuration = 60`.

2. `POST` handler:
   - Auth guard: use `createSupabaseServerClient()` to get user, then check role via service client `profiles` table. Require `isAdminOrAbove(role)`. Return 403 if not admin.
   - Parse body: `{ question: string }`. Validate question is non-empty string. Return 400 if invalid.
   - Load chatbot_config (same pattern as chat-service.ts): `getSupabaseService().from('chatbot_config').select('*').single()`.
   - Load and decrypt the API key: `admin_secrets` table using `config.selected_key_id`.
   - Use a FAQ-specific system prompt: `"You are generating a concise, helpful FAQ answer for the GOYA yoga association platform. Answer the following user question clearly and professionally, in 2-4 sentences."`
   - Call the AI provider (OpenAI or Anthropic, based on `secretRow.provider`) with streaming enabled.
   - Stream NDJSON response: emit `{ type: 'token', content: token }` chunks, then `{ type: 'done' }` at end.
   - Use same `encodeChunk` pattern: `new TextEncoder().encode(JSON.stringify(data) + '\n')`.
   - On error, emit `{ type: 'error', message: string }`.
   - Return `new Response(stream, { status: 200, headers: { 'Content-Type': 'text/plain; charset=utf-8' } })`.

Import requirements:
```typescript
import { createSupabaseServerClient } from '@/lib/supabaseServer'
import { getSupabaseService } from '@/lib/supabase/service'
import { decrypt } from '@/lib/secrets/encryption'
import { isAdminOrAbove } from '@/lib/roles'
import OpenAI from 'openai'
import Anthropic from '@anthropic-ai/sdk'
```

**In `app/admin/inbox/actions.ts` — add two new server actions:**

1. `addToFaq(ticketId: string, question: string, answer: string, adminUserId: string)`:
   - Auth guard (same pattern as other actions in this file).
   - Check for duplicate FAQ: query `faq_items` where `question` ILIKE the provided question. If match found, return `{ success: false, error: 'duplicate', existingId: match.id }` (special error code so UI can show warning).
   - Insert into `faq_items` with `{ question, answer, status: 'published', created_by: adminUserId }`. Note: use status `'published'` directly — per CONTEXT.md decision, "Add to FAQ creates published entry directly, no draft step".
   - Update ticket: set `status: 'resolved'`, `resolved_at: now`, `resolved_by: adminUserId`.
   - Revalidate `/admin/inbox` and `/admin/chatbot`.
   - Return `{ success: true }`.

2. `rejectUnanswered(ticketId: string, rejectionReason: string, adminUserId: string)`:
   - Auth guard.
   - Validate rejectionReason is non-empty.
   - Update ticket: set `status: 'resolved'`, `rejection_reason: rejectionReason`, `resolved_at: now`, `resolved_by: adminUserId`.
   - Revalidate `/admin/inbox`.
   - Return `{ success: true }`.

3. `addToFaqForce(ticketId: string, question: string, answer: string, adminUserId: string)`:
   - Same as `addToFaq` but skips the duplicate check. Called when admin confirms they want to proceed despite duplicate warning.

IMPORTANT: Use column name `question_summary` when reading from `support_tickets`, but `question` when writing to `faq_items` (different tables, different column names).

**Create `app/admin/inbox/UnansweredPanel.tsx`:**

A `'use client'` component that provides the full unanswered question resolution workflow.

Props:
```typescript
interface UnansweredPanelProps {
  ticket: SupportTicket
  adminUserId: string
  onResolved: (updatedTicket: SupportTicket) => void
}
```

State:
- `mode`: `'idle' | 'generating' | 'editing' | 'rejecting'`
- `generatedAnswer`: string (accumulated from stream)
- `editedAnswer`: string (textarea value, initialized from generatedAnswer)
- `rejectionReason`: string
- `saving`: boolean
- `error`: string | null
- `duplicateWarning`: boolean (set when addToFaq returns 'duplicate' error)

**"Mattea answers this" button:**
- When clicked, set `mode = 'generating'`.
- Fetch `POST /api/chatbot/generate-answer` with `{ question: ticket.question_summary }`.
- Read NDJSON stream: on each `{ type: 'token', content }` chunk, append to `generatedAnswer`.
- On `{ type: 'done' }`, set `mode = 'editing'` and copy `generatedAnswer` into `editedAnswer`.
- On `{ type: 'error' }`, show error, set `mode = 'idle'`.
- While generating, show the question in a read-only block and the answer streaming below it (like a mini chat). Show a pulsing dot or "Generating..." indicator.

**Editing mode:**
- Show ticket.question_summary in a read-only block at top.
- Show `editedAnswer` in a `<textarea>` (rows=6, resizable).
- Two buttons: "Add to FAQ" (primary, teal bg-[#00B5A3]) and "Cancel" (text button, returns to idle).
- On "Add to FAQ": call `addToFaq(ticket.id, ticket.question_summary, editedAnswer, adminUserId)`.
  - If result is `{ success: false, error: 'duplicate' }`: set `duplicateWarning = true`.
  - If success: call `onResolved` with updated ticket (status: 'resolved').
- If `duplicateWarning` is true, show amber warning: "A similar question already exists in the FAQ. Add anyway?" with "Yes, add" button that calls `addToFaqForce` and "Cancel" button.

**"Mattea won't answer this" button:**
- When clicked, set `mode = 'rejecting'`.
- Show a dropdown/select with reasons: "Out of scope (not yoga related)", "Sensitive / inappropriate", "Duplicate of existing FAQ", "Other".
- Plus a "Confirm rejection" button and "Cancel".
- On confirm: call `rejectUnanswered(ticket.id, rejectionReason, adminUserId)`. On success, call `onResolved` with updated ticket (status: 'resolved', rejection_reason: rejectionReason).

**Layout:**
- Render as a card section below the conversation in TicketViewer (border-t, bg-amber-50/30 background to distinguish).
- In idle mode, show two buttons side by side: "Mattea answers this" (teal) and "Mattea won't answer this" (slate border).
- Use Sparkles icon from lucide-react for "Mattea answers this" and XCircle for "won't answer".

**Wire into TicketViewer.tsx:**

1. Import `UnansweredPanel` and `SupportTicket` type.
2. Between the conversation section and the reply section, add a conditional render:
```tsx
{ticket.ticket_type === 'unanswered_question' && ticket.status !== 'resolved' && (
  <UnansweredPanel
    ticket={ticket}
    adminUserId={adminUserId}
    onResolved={(updated) => onStatusChange(updated)}
  />
)}
```
3. For resolved unanswered tickets, show a read-only summary:
```tsx
{ticket.ticket_type === 'unanswered_question' && ticket.status === 'resolved' && (
  <div className="px-5 py-3 border-t border-[#E5E7EB] bg-green-50/50">
    <p className="text-sm text-green-700 font-medium">
      {ticket.rejection_reason ? `Rejected: ${ticket.rejection_reason}` : 'Resolved — added to FAQ'}
    </p>
  </div>
)}
```

Style conventions: Use project color palette (#1B3A5C for headings, #374151 for body text, #00B5A3 for primary actions, slate borders #E5E7EB). Text sizes: text-sm for body, text-xs for labels.
