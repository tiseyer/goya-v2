    **1. Create `app/api/chatbot/conversations/[id]/feedback/route.ts`:**

    Export a PATCH handler. Extract `id` from params (Next.js 15 async params pattern: `const { id } = await params`). Parse request body as `{ feedback: 'helpful' | 'not_helpful' }`.

    Map values: `'helpful'` -> `'up'`, `'not_helpful'` -> `'down'` (DB column `user_feedback` uses `'up'`/`'down'` per types.ts ChatSession).

    Use `getSupabaseService()` cast to `any` (matching project pattern) to update `chat_sessions`:
    ```typescript
    const { error } = await (getSupabaseService() as any)
      .from('chat_sessions')
      .update({ user_feedback: mapped, feedback_at: new Date().toISOString() })
      .eq('id', id)
    ```

    Return 200 `{ success: true }` on success, 400 for invalid body, 500 on DB error.

    No auth check needed per CONTEXT.md decision ("No auth required beyond being the session owner"). The session ID itself is the access token.

    **2. Create `app/components/chat/FeedbackButtons.tsx`:**

    A `'use client'` component with props:
    ```typescript

      sessionId: string | null

    }
    ```

    Component manages its own state: `feedback: 'helpful' | 'not_helpful' | null` and `submitting: boolean`.

    Render two small icon buttons using Lucide `ThumbsUp` and `ThumbsDown` (already available in project via lucide-react).

    Styling per CONTEXT.md decision:
    - Container: `flex gap-1.5 mt-1.5 opacity-0 group-hover:opacity-60 hover:!opacity-100 transition-opacity` (shows on hover of parent message bubble)
    - When `visible` is false: render nothing (return null)
    - Each button: `p-1 rounded hover:bg-slate-100 dark:hover:bg-slate-800 transition-colors disabled:cursor-default`
    - Icon size: `w-3.5 h-3.5`
    - Default color: `text-slate-400`
    - ThumbsUp selected: `text-green-500` with fill
    - ThumbsDown selected: `text-red-400` with fill
    - After selection: disable both buttons, show micro-confirmation text "Thanks!" in `text-xs text-slate-400` that fades in

    On click, call:
    ```typescript

      headers: { 'Content-Type': 'application/json' },

    ```

    If `sessionId` is null, disable buttons (grey out). Handle fetch errors silently (non-fatal).
