<task type="auto">
  <name>Task 2: Create WordPress media migration script</name>
  <files>scripts/migrate-wp-media.ts</files>
  <action>
Create `scripts/migrate-wp-media.ts` following the exact patterns from `scripts/migrate-wp-avatars.ts` for dotenv loading, Supabase client creation, and --dry-run flag support.

**Setup (copy pattern from migrate-wp-avatars.ts):**
- dotenv config with both cwd and __dirname fallback for .env.local
- createClient with NEXT_PUBLIC_SUPABASE_URL + SUPABASE_SERVICE_ROLE_KEY
- isDryRun flag from process.argv
- Support `--resume` flag to continue from last saved page

**WP REST API configuration:**
- Base URL: `https://members.globalonlineyogaassociation.org`
- Endpoint: `/wp-json/wp/v2/media?per_page=100&page={N}`
- Auth: Basic Auth header with credentials `WP-Media-Library-Migration` / `hUfe NTr9 tk1z sCcE qTnp CnvB` (base64 encode `username:password`)
- Paginate until API returns empty array or 400 status (WP returns 400 on past-last page)

**Author matching:**
- WP media items include `author` (numeric WP user ID)
- For each unique author, fetch `/wp-json/wp/v2/users/{author_id}` to get their email
- Cache WP user ID -> email mapping in memory (Map<number, string>)
- Look up Supabase profile by email: `supabase.from('profiles').select('id').eq('email', wpUserEmail).maybeSingle()`
- Cache email -> Supabase profile ID mapping (Map<string, string | null>)
- If no matching profile found, set uploaded_by to null

**File processing for each WP media item:**
- Skip if `source_url` contains `wp-content/uploads/avatars/` (avatar paths handled by separate script)
- Extract original file URL from `source_url` field
- Determine mime type from `mime_type` field on the WP media object
- Determine subfolder from mime type: `images/` for image/*, `documents/` for application/pdf, `video/` for video/*, `audio/` for audio/*, `other/` for anything else
- Download the file via fetch
- Generate storage path: `wp-media/{subfolder}/{wp_media_id}_{original_filename}` (prefix with WP ID to avoid collisions)
- Upload to `media` bucket in Supabase Storage with upsert: true
- Get public URL via `supabase.storage.from('media').getPublicUrl(path)`

**Database upsert:**
- Upsert into media_items using onConflict: 'wp_media_id' for deduplication
- Fields to set:
  - bucket: 'media'
  - folder: null (no folder FK for migrated media)
  - file_name: original filename from WP
  - file_path: the storage path used above
  - file_url: public URL from Storage
  - file_type: mime_type from WP
  - file_size: byte length of downloaded buffer
  - uploaded_by: matched Supabase profile ID or null
  - alt_text: from WP media `alt_text` field
  - caption: from WP media `caption.rendered` field (strip HTML tags)
  - title: from WP media `title.rendered` field (strip HTML tags)
  - width: from WP media `media_details.width` if present
  - height: from WP media `media_details.height` if present
  - wp_media_id: the WP media item ID (number)

**Progress tracking:**
- Create `.migration-state/` directory if not exists
- After each page completes, write `.migration-state/wp-media-progress.json`:
  ```json
  {
    "lastCompletedPage": N,
    "totalProcessed": M,
    "totalSkipped": S,
    "totalFailed": F,
    "lastUpdated": "ISO timestamp"
  }
  ```
- On `--resume`, read progress file and start from lastCompletedPage + 1

**Failure tracking:**
- On each item failure, append to in-memory failures array
- After each page, write `.migration-state/wp-media-failures.json`:
  ```json
  [{ "wp_media_id": N, "source_url": "...", "error": "...", "timestamp": "..." }]
  ```

**Rate limiting and resilience:**
- 100ms delay between page fetches (`await new Promise(r => setTimeout(r, 100))`)
- Retry failed downloads up to 3 times with exponential backoff (1s, 2s, 4s)
- On retry exhaustion, log to failures and continue to next item
- Wrap individual item processing in try/catch so one failure doesn't stop the batch

**Summary output:**
- At end, print summary: total processed, uploaded, skipped (avatars), skipped (already exists), failed
- Print path to failures JSON if any failures occurred

**IMPORTANT:** Do NOT run the script. Only create the file.
