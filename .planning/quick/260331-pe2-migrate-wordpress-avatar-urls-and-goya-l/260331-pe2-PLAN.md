  <action>
Create scripts/migrate-wp-avatars.ts following the exact env loading and supabase client pattern from scripts/backfill-media-items.ts.

Parse CLI args: check for --dry-run flag via process.argv.includes('--dry-run').

**Avatar migration function (migrateAvatars):**
1. Query profiles where avatar_url LIKE '%members.globalonlineyogaassociation.org%'
2. Process in batches of 50 using .range(offset, offset+49)
3. For each profile:
   - Skip if avatar_url already contains the project's Supabase storage URL (resumability)
   - Fetch the image from the WordPress URL using global fetch (Node 18+)
   - Determine file extension from Content-Type header or URL (default to 'jpg')
   - Convert response to ArrayBuffer then Buffer
   - Upload to 'avatars' bucket with path: `{user_id}/avatar.{ext}` using supabase.storage.from('avatars').upload() with upsert: true
   - Get public URL via supabase.storage.from('avatars').getPublicUrl()
   - Update profiles.avatar_url with new URL
   - Insert into media_items: bucket='avatars', file_name=`avatar.{ext}`, file_path=`{user_id}/avatar.{ext}`, file_url=new public URL, file_type from Content-Type, file_size from buffer length, uploaded_by=user_id
   - Log: "Migrated user {id}: {old_url} -> {new_url}"
   - On error (fetch fail, upload fail): log error with user id and continue, increment failed counter
4. In dry-run mode: log what would happen but skip fetch/upload/update steps

**Logo migration function (migrateLogos):**
1. Define array of logo files: ['Favicon.png', 'GOYA Logo Black.png', 'GOYA Logo Blue.png', 'GOYA Logo Short.png', 'GOYA Logo White.png']
2. For each file:
   - Read from public/images/{filename} using fs.readFileSync
   - Check if already exists in media_items with bucket='uploads' and file_path=`brand/{filename}` (resumability)
   - Upload to 'uploads' bucket with path: `brand/{filename}` using upsert: true
   - Get public URL
   - Insert into media_items: bucket='uploads', folder='brand', file_name=filename, file_path=`brand/{filename}`, file_url=public URL, file_type='image/png', file_size from buffer length
   - Log progress
3. In dry-run mode: log what would happen but skip upload/insert

**Main function:**
- Run migrateAvatars() then migrateLogos()
- Print summary: total migrated, total failed, total skipped for avatars; total uploaded for logos
- Use try/catch with process.exit(1) on fatal error (same pattern as backfill script)

Use inferMimeType helper (copy from backfill-media-items.ts) for avatar content type fallback.
