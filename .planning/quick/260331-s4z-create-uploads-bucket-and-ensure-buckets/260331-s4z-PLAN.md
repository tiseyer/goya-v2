Create `scripts/ensure-buckets.ts` that:
1. Loads env from `.env.local` using the same dotenv pattern as `migrate-wp-avatars.ts`
   (resolve from cwd + fallback from __dirname)
2. Creates a service-role Supabase client inline (same as other scripts — NOT importing from lib/)
3. Defines a BUCKETS array:
   - { name: 'avatars', public: true }
   - { name: 'uploads', public: true }
   - { name: 'school-documents', public: false }
4. For each bucket: call `supabase.storage.createBucket(name, { public })`.
   If error.message includes 'already exists', log "Bucket '{name}' already exists" and continue.
   On success, log "Created bucket '{name}' (public: {public})".
   On any other error, log it and set exit code 1.
5. At end, log summary: "Done. {created} created, {existing} already existed, {failed} failed."

Add npm script to package.json:
  "buckets:ensure": "tsx scripts/ensure-buckets.ts"
  </action>
