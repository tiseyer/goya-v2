  <action>
The subscriptions table has a `user_id` column (UUID referencing auth.users, which equals profiles.id). Currently the page fetches profile data via stripe_customer_id but does not pass user_id through to the table for linking.

**page.tsx changes:**
1. Add `user_id` to the RawSubscription type (user_id: string | null)
2. In the merge step, pass `userId: s.user_id ?? null` into each SubscriptionRow
3. Also fetch profiles via user_id (not just stripe_customer_id) as a fallback — query profiles.id IN (userIds) for any subs where stripe_customer_id is null but user_id exists. Merge both profile lookups into profileMap keyed by both stripe_customer_id and user_id.

**SubscriptionsTable.tsx changes:**
1. Add `userId: string | null` to SubscriptionRow type
2. Customer column: wrap customer name in a Link to `/admin/users/${sub.userId}` when userId is not null. If userId is null, render plain text as before. Import Link from 'next/link'.
3. Amount column: change from `formatCurrency(sub.amount, sub.currency)` to `formatCurrency(sub.amount, sub.currency) + '/' + sub.interval` — e.g. "$39.00/year" or "$9.99/month".
4. Stripe ID column: change truncation from `sub.stripeSubscriptionId.slice(0, 8)` to show `sub_` prefix + first 4 chars after prefix + `...` + last 3 chars. Example: "sub_1ABC...XYZ". Use: `const sid = sub.stripeSubscriptionId; sid.length > 15 ? sid.slice(0, 8) + '...' + sid.slice(-3) : sid`.
5. Status badges: update STATUS_PILL to match task spec colors:
   - active: 'bg-emerald-100 text-emerald-700' (green - keep as-is)
   - canceled: 'bg-red-100 text-red-700' (red - change from slate)
   - past_due: 'bg-red-100 text-red-700' (red - change from amber)
   - trialing: 'bg-blue-100 text-blue-700' (keep as-is)
   - incomplete: 'bg-yellow-100 text-yellow-700' (yellow for pending-like)
   - paused: 'bg-amber-100 text-amber-700' (amber for on-hold-like)
   - unpaid: 'bg-red-100 text-red-700' (keep as-is)

**route.ts changes:**
1. Add `user_id` to RawSubscription type
2. Add `userId: s.user_id ?? null` to SubscriptionRow type and mapping
3. Same dual profile lookup logic as page.tsx (by stripe_customer_id and by user_id)
