# Quick Task 260403-c47: Nav Skeleton Loading State Summary

**One-liner:** Nav skeleton pills/bars show during auth loading, eliminating logged-out-then-logged-in pop-in flash.

## What Was Done

Added skeleton loading states to both the desktop and mobile navigation sections in `app/components/Header.tsx` so that nav items no longer flash logged-out links before switching to logged-in links when auth state resolves.

## Changes Made

### Desktop nav (`hidden lg:flex` section, ~line 1027)

Wrapped the entire nav link block in an `authLoading` ternary:
- **Loading:** 4 skeleton pill divs (`h-[34px] rounded-lg bg-slate-100 animate-pulse`) with widths [80, 64, 72, 64]px
- **Resolved:** Dashboard link (if logged in) + the appropriate NAV_LINKS array, all rendered at once

### Mobile nav (hamburger overlay, ~line 1165)

Wrapped the link section in an `authLoading` ternary:
- **Loading:** 4 skeleton bar divs (`h-10 rounded-lg bg-slate-100 animate-pulse`) with widths [128, 96, 112, 96]px
- **Resolved:** Dashboard link + nav links + MobileCartToggle (if logged in), all rendered at once

The right-side auth area (lines ~1056-1107) and mobile avatar/login button were left unchanged — they already had proper skeleton handling.

## Deviations from Plan

None — plan executed exactly as written.

The `MobileCartToggle` component was already inside the mobile nav block; it was correctly moved into the resolved (non-loading) branch as part of the refactor, which is the correct behavior (cart should not show during loading).
