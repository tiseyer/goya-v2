**A. Create `app/schools/create/actions.ts`** (per REG-03):

Add `'use server'` directive at top. Import `createSupabaseServerActionClient` from `@/lib/supabaseServer`, `getStripe` from `@/lib/stripe/client`, `redirect` from `next/navigation`.

Export `createSchoolCheckoutSession(schoolName: string, slug: string, designationTypes: string[])` returning `Promise<{ url: string } | { error: string }>`:

1. Get user via `supabase.auth.getUser()`. If no user, `redirect('/sign-in')`.
2. Validate: `designationTypes.length > 0` or return error.
3. Read env vars `STRIPE_SCHOOL_ANNUAL_PRICE_ID` and `STRIPE_SCHOOL_SIGNUP_PRICE_ID` (2 shared price IDs since all designations have identical pricing at EUR 40/year + EUR 99 signup). If either is missing, return `{ error: 'Stripe school prices not configured' }`.
4. Build `line_items`: one entry per designation with `{ price: STRIPE_SCHOOL_ANNUAL_PRICE_ID, quantity: 1 }`. NOTE: For multiple designations, add multiple line items (one per designation).
5. Build `subscription_data.add_invoice_items`: one entry per designation with `{ price: STRIPE_SCHOOL_SIGNUP_PRICE_ID }` for the one-time signup fee.
6. Call `getStripe().checkout.sessions.create()` with:
   - `mode: 'subscription'`
   - `line_items` from step 4
   - `subscription_data: { add_invoice_items: [...], metadata: { type: 'school_registration', user_id: user.id } }`
   - `metadata: { type: 'school_registration', user_id: user.id, school_name: schoolName, school_slug: slug, designation_types: JSON.stringify(designationTypes) }`
   - `success_url: \`${process.env.NEXT_PUBLIC_APP_URL}/schools/create/success?session_id={CHECKOUT_SESSION_ID}&slug=${slug}\``
   - `cancel_url: \`${process.env.NEXT_PUBLIC_APP_URL}/schools/create?step=2\``
7. Return `{ url: session.url! }` on success, `{ error: message }` on catch.

1. Extract from `session.metadata`: `user_id`, `school_name`, `school_slug`, `designation_types` (JSON.parse).

3. Use `getSupabaseService()` (service role, same pattern as existing handler).
4. Insert school: `.from('schools').insert({ owner_id: user_id, name: school_name, slug: school_slug, status: 'pending' }).select('id').single()`
5. If insert error with code '23505' (unique violation on slug or owner), log and return `{ status: 'processed' }` (idempotent — already created).
6. Insert school_designations: For each type in `designation_types`, insert `{ school_id: school.id, designation_type: type, stripe_subscription_id: subscription_id, stripe_price_id: process.env.STRIPE_SCHOOL_ANNUAL_PRICE_ID, signup_fee_paid: true, signup_fee_amount: 9900, annual_fee_amount: 4000 }`.
7. Update profile: `.from('profiles').update({ principal_trainer_school_id: school.id }).eq('id', user_id)`.
8. Notify admins (same pattern as existing handler): query admins, insert notifications with `type: 'school_registration_submitted'`, `title: 'New School Registration'`, `body: \`${school_name} has been registered and is pending review.\``, `link: '/admin/inbox'`, `actor_id: user_id`.
9. Return `{ status: 'processed' }`.

Use `// eslint-disable-next-line @typescript-eslint/no-explicit-any` before Supabase calls that need it (same pattern as existing handler).
