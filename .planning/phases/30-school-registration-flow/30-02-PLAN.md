**B. Replace `app/schools/create/page.tsx` with a server component + client wizard:**

The page.tsx is a SERVER component that:
1. Auth-gates: get user via `createSupabaseServerClient()`, redirect to `/sign-in` if no user.
2. Role-gates: query `profiles` table for user's `role` and `principal_trainer_school_id`. If `role !== 'teacher'`, redirect to `/dashboard`. If `principal_trainer_school_id` is not null, redirect to `/dashboard` (already has a school).
3. Fetches designation products: query `products` table `.select('slug, name, full_name, image_path, price_cents').eq('category', 'school_designation').order('priority')`.
4. Renders `<Suspense fallback={<div className="animate-pulse">Loading...</div>}>` wrapping a client component `<SchoolCreateWizard products={products} />`.
5. Uses `PageContainer` as the outer wrapper.

**C. Create the `SchoolCreateWizard` client component** (can be defined in the same file or as a separate component — keep it in the same file for simplicity since it is tightly coupled to this page):

Add `'use client'` section. The wizard reads step from URL: `useSearchParams().get('step')` defaulting to '1'. Use `useRouter()` for navigation.

**Step indicator:** A simple 3-step progress bar at the top showing "School Name", "Designations", "Payment". Current step highlighted with brand color `#1B3A5C`. Completed steps show a checkmark.

**Step 1 — School Name (per REG-01):**
- Heading: "Name Your School"
- Text input for school name (required, min 3 chars)
- Auto-generated slug shown below in a read-only-looking input that IS editable. Format: `goya.org/schools/{slug}`. Use `generateSlug(name)` from `@/lib/schools/slug` on name change.
- Debounced uniqueness check (500ms) on slug change: `fetch(\`/api/schools/check-slug?slug=${slug}\`)`. Show green check if available, red X if taken.
- "Continue" button enabled only when: name.length >= 3 AND slug.length >= 1 AND slug is available (not loading, not taken).
- Store name + slug in React state.

**Step 2 — Select Designations (per REG-02):**
- Heading: "Choose Your Designations"
- Subtext: "Select the designations your school will offer. Each designation costs EUR 40.00/year plus a one-time EUR 99.00 signup fee."
- Render products (passed as props) as selectable cards in a responsive grid (2 cols on md, 1 on mobile).
- Each card shows: product image (using `next/image` with the `image_path`), product name (e.g., "GOYA-CYS200"), full name below, a checkbox/selected state indicator.
- Cards have a border highlight and subtle background when selected (e.g., `border-[#1B3A5C] bg-blue-50`).
- Multi-select: clicking toggles selection. Track selected designation types in state as string[] (extract type from slug: e.g., 'goya-cys200' -> 'CYS200', by taking the part after 'goya-' and uppercasing).
- **Running total** at the bottom: "Selected: {N} designation(s) — EUR {N * 40}.00/year + EUR {N * 99}.00 signup fee". Show both annual and one-time totals separately.
- Below the total: "Can't find your specialty? You can add more designations later." in muted text.
- "Continue to Payment" button enabled only when at least 1 designation selected.
- "Back" link to go back to step 1 (preserving name/slug state).

**Step 3 — Payment redirect:**
- When user clicks "Continue to Payment" on step 2, call the `createSchoolCheckoutSession` server action with (schoolName, slug, selectedDesignationTypes).
- Show a loading state ("Redirecting to payment...") while the action executes.
- If result has `url`, do `window.location.href = result.url` (full redirect to Stripe).
- If result has `error`, show an error message with a "Try Again" button.

**State persistence:** Store wizard form data (name, slug, selectedDesignations) in `localStorage` under key `'school-registration-draft'` so state survives page refresh or Stripe cancel_url return. On mount, restore from localStorage if present. Clear localStorage on successful redirect to Stripe.

**Styling:**
- Use the project's brand colors: primary `#1B3A5C`, accents from existing pages.
- Cards should have `rounded-xl border border-[#E5E7EB]` default, `border-[#1B3A5C] bg-blue-50/50` when selected.
- Buttons: primary `bg-[#1B3A5C] text-white rounded-lg px-6 py-3 font-semibold`, disabled `opacity-50 cursor-not-allowed`.
