### Layout Pattern

The Third Party Keys tab splits into two vertically stacked sections:

```
┌─────────────────────────────────────────────────┐
│  AI Providers                    [+ Add AI Provider Key] │
│  ─────────────────────────────────────────────── │
│  table: Display Name | Provider | Model | Added | Actions │
└─────────────────────────────────────────────────┘

(mt-8 gap)

┌─────────────────────────────────────────────────┐
│  [All][Auth][Analytics][Payments][Other]  [search] [+ Add Secret] │
│  table: Name | Category | Description | Last Updated | Actions │
└─────────────────────────────────────────────────┘
```

AI Providers section renders ABOVE the existing general secrets section. Section heading uses `text-base font-semibold text-[#1B3A5C]`.

### Focal Point

The primary focal point of the screen is the **AI Providers section heading row** — specifically the "Add AI Provider Key" button in the top-right of that section. This is the primary action users arrive to perform. The section heading `text-2xl font-semibold text-[#1B3A5C]` at the top of the tab anchors the eye before scanning down to the table.

## Interaction States

### AI Provider Key Table

| State | Visual |
|-------|--------|
| Empty (no AI keys) | Lock icon + "No AI provider keys yet. Add one to power the chatbot." |
| Row hover | `hover:bg-slate-50` |
| Delete pending confirmation | Inline "Sure?" + red Delete button + "Keep Key" — identical to SecretsTab pattern except cancel label |
| Saving (modal submit) | Button shows "Saving..." with `disabled:opacity-50` |

### AI Provider Modal

| Field | Behavior |
|-------|---------|
| Provider select | Dropdown: OpenAI, Anthropic. On change, repopulates Model dropdown |
| Model dropdown | Populated client-side on provider select — no API call |
| API Key (value) | `type="password"`, `autoComplete="new-password"`. Required on create, blank = keep current on edit |
| Display Name | Text input, optional. Placeholder: "e.g. Production OpenAI Key" |
| Edit mode key field | Placeholder: "Leave blank to keep current key" |

### Post-Save Security Constraint

After saving an AI provider key, the raw key value is NEVER displayed. The table shows display name, provider, model, and created date only. This is enforced by the same masked-value pattern used in the general secrets table (KEYS-04).
