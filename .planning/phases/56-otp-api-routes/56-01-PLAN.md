    - Insert new code row, send email via sendEmailFromTemplate with templateKey 'device_verification_otp', variables { code: plaintext_code_as_string, deviceName: goya_device_name cookie value or 'your device', firstName: '' }

    - On any DB error → return 500 { error: 'internal_error' }

  <action>
    Create app/api/device-verification/send/route.ts as a Next.js App Router POST handler.

    Import: NextRequest, NextResponse from 'next/server'; crypto from 'node:crypto'; getSupabaseService from '@/lib/supabase/service'; sendEmailFromTemplate from '@/lib/email/send'.

    Do NOT export `runtime` — this must stay on the Node.js runtime for crypto.

    Step 1 — Parse cookie:
    Read `device_pending_verification` from request.cookies. Split on '|'. If parts.length !== 2 or either part is empty, return 400 { error: 'missing_pending_cookie' }.
    Assign profileId = parts[0], deviceFingerprint = parts[1].

    Step 2 — Check for existing valid code (idempotency):
    Query device_verification_codes where profile_id = profileId AND device_fingerprint = deviceFingerprint AND invalidated = false AND expires_at > now().
    Order by created_at desc, limit 1.
    If found:
      - If created_at is within the last 2 minutes (Date.now() - new Date(row.created_at).getTime() < 120_000) → return 200 { expiresAt: row.expires_at }
      - Otherwise → invalidate it: update set invalidated=true where id = row.id (continue to generate new code below)

    Step 3 — Generate new OTP:
    const code = crypto.randomInt(100000, 999999)
    const codeStr = String(code)
    const hashed_code = crypto.createHash('sha256').update(codeStr).digest('hex')
    const expires_at = new Date(Date.now() + 10 * 60 * 1000).toISOString()

    Step 4 — Insert new code row:
    Insert { profile_id: profileId, device_fingerprint: deviceFingerprint, hashed_code, expires_at } into device_verification_codes.
    If insert error → return 500 { error: 'internal_error' }.

    Step 5 — Fetch profile email:
    Query profiles where id = profileId, select email. If not found → return 500 { error: 'profile_not_found' }.

    Step 6 — Send email:
    const deviceName = request.cookies.get('goya_device_name')?.value ?? 'your device'
    Call sendEmailFromTemplate({ to: profile.email, templateKey: 'device_verification_otp', variables: { code: codeStr, deviceName } }).
    Email send failure is non-fatal — log it but still return 200 so the code is usable.

    Step 7 — Return 200 { expiresAt: expires_at }.
  </action>

  <files>app/api/device-verification/verify/route.ts</files>
  <behavior>
    - If device_pending_verification cookie is missing or malformed → return 400 { error: 'missing_pending_cookie' }
    - If request body has no `code` field or code is not a 6-digit string → return 400 { error: 'invalid_code_format' }
    - If no non-invalidated code exists for (profile_id, device_fingerprint) → return 400 { error: 'no_pending_verification' }
    - If code is expired → return 400 { error: 'code_expired' }
    - If attempt_count >= 5 OR invalidated is already true → return 400 { error: 'too_many_attempts' } WITHOUT performing comparison
    - Code comparison: crypto.timingSafeEqual(Buffer.from(submittedHash, 'hex'), Buffer.from(storedHash, 'hex')) where submittedHash = crypto.createHash('sha256').update(submittedCode).digest('hex')
    - On mismatch: increment attempt_count by 1; if new count >= 5, also set invalidated=true; return 400 { error: 'invalid_code', remainingAttempts: Math.max(0, 5 - newCount) }
    - On match: (1) insert trusted_devices row with profile_id, device_fingerprint, device_name from cookie, ip_address from x-forwarded-for header; (2) set invalidated=true on the code row; (3) clear device_pending_verification cookie; return 200 { success: true }
    - Route MUST NOT export runtime = 'edge'
  </behavior>
  <action>
    Create app/api/device-verification/verify/route.ts as a Next.js App Router POST handler.

    Import: NextRequest, NextResponse from 'next/server'; crypto from 'node:crypto'; getSupabaseService from '@/lib/supabase/service'.

    Do NOT export `runtime` — this must stay on the Node.js runtime for crypto.timingSafeEqual.

    Step 1 — Parse pending cookie:
    Read `device_pending_verification` from request.cookies. Split on '|'. Validate 2 non-empty parts. Return 400 { error: 'missing_pending_cookie' } if invalid.
    Assign profileId = parts[0], deviceFingerprint = parts[1].

    Step 2 — Parse and validate request body:
    const body = await request.json(). Extract body.code as string.
    If !code or !/^\d{6}$/.test(code) → return 400 { error: 'invalid_code_format' }.

    Step 3 — Fetch latest active code:
    Query device_verification_codes where profile_id = profileId AND device_fingerprint = deviceFingerprint AND invalidated = false, order by created_at desc, limit 1.
    If not found → return 400 { error: 'no_pending_verification' }.
    If row.expires_at < now → return 400 { error: 'code_expired' }.
    If row.attempt_count >= 5 → return 400 { error: 'too_many_attempts' }.

    Step 4 — Compare code:
    const submittedHash = crypto.createHash('sha256').update(code).digest('hex')
    const storedHash = row.hashed_code
    Use crypto.timingSafeEqual(Buffer.from(submittedHash, 'hex'), Buffer.from(storedHash, 'hex')) — both buffers are 32 bytes (64 hex chars). Wrap in try/catch; on error treat as mismatch.

    Step 5a — On MISMATCH:
    const newCount = row.attempt_count + 1
    Update device_verification_codes set attempt_count = newCount, and if newCount >= 5 also set invalidated = true, where id = row.id.
    Return 400 { error: 'invalid_code', remainingAttempts: Math.max(0, 5 - newCount) }.

    Step 5b — On MATCH:
    (i) Insert trusted_devices: { profile_id: profileId, device_fingerprint: deviceFingerprint, device_name: request.cookies.get('goya_device_name')?.value ?? null, ip_address: request.headers.get('x-forwarded-for')?.split(',')[0].trim() ?? null }
    (ii) Update device_verification_codes set invalidated = true where id = row.id.
    (iii) Build response: const response = NextResponse.json({ success: true })
    response.cookies.set('device_pending_verification', '', { path: '/', maxAge: 0, httpOnly: true, sameSite: 'lax', secure: process.env.NODE_ENV === 'production' })
    return response.
    On any DB error in step 5b → return 500 { error: 'internal_error' }.
  </action>
