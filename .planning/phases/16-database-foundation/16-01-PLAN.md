```sql
ALTER TABLE events ADD COLUMN IF NOT EXISTS event_type text NOT NULL DEFAULT 'goya' CHECK (event_type IN ('goya', 'member'));
```

2. Add `created_by` FK to events:
```sql
ALTER TABLE events ADD COLUMN IF NOT EXISTS created_by uuid REFERENCES profiles(id) ON DELETE SET NULL;
```

3. Extend `status` column — the current CHECK constraint allows 'published', 'draft', 'cancelled', 'deleted'. Need to drop and recreate to add 'pending_review' and 'rejected':
```sql
ALTER TABLE events DROP CONSTRAINT IF EXISTS events_status_check;
ALTER TABLE events ADD CONSTRAINT events_status_check CHECK (status IN ('draft', 'pending_review', 'published', 'rejected', 'cancelled', 'deleted'));
```

4. Add `rejection_reason` column:
```sql
ALTER TABLE events ADD COLUMN IF NOT EXISTS rejection_reason text;
```

5. Create `event_audit_log` table:
```sql
CREATE TABLE IF NOT EXISTS event_audit_log (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  event_id uuid NOT NULL REFERENCES events(id) ON DELETE CASCADE,
  action text NOT NULL CHECK (action IN ('created', 'edited', 'status_changed', 'deleted')),
  performed_by uuid REFERENCES profiles(id) ON DELETE SET NULL,
  performed_by_role text,
  changes jsonb,
  created_at timestamptz NOT NULL DEFAULT now()
);
```

6. Create index on event_audit_log for event_id lookups:
```sql
CREATE INDEX IF NOT EXISTS idx_event_audit_log_event_id ON event_audit_log(event_id);
```

### Task 2: Create RLS policies

<read_first>
- supabase/migrations/20260322_add_events_table.sql (existing RLS patterns)
- supabase/migrations/20260348_api_keys.sql (created_by RLS pattern)
</read_first>

<action>
Create migration file `supabase/migrations/20260371_member_events_rls.sql` with:

1. Enable RLS on event_audit_log:
```sql
ALTER TABLE event_audit_log ENABLE ROW LEVEL SECURITY;
```

2. Events RLS — Teachers/WPs/School owners INSERT own events:
```sql
CREATE POLICY "members_insert_own_events" ON events
  FOR INSERT TO authenticated
  WITH CHECK (
    created_by = auth.uid()
    AND event_type = 'member'
    AND EXISTS (
      SELECT 1 FROM profiles WHERE id = auth.uid() AND role IN ('teacher', 'wellness_practitioner', 'admin')
    )
  );
```
Note: 'school' role doesn't exist in the enum. School owners are teachers who own schools. Use teacher/WP/admin.

3. Events RLS — Members SELECT own events (all statuses except deleted):
```sql
CREATE POLICY "members_select_own_events" ON events
  FOR SELECT TO authenticated
  USING (
    (created_by = auth.uid() AND status != 'deleted')
    OR (status = 'published')
    OR EXISTS (
      SELECT 1 FROM profiles WHERE id = auth.uid() AND role IN ('moderator', 'admin')
    )
  );
```
This unified SELECT policy handles: members see own non-deleted + all see published + mods/admins see all non-deleted.

4. Admin SELECT (including deleted):
```sql
CREATE POLICY "admins_select_all_events" ON events
  FOR SELECT TO authenticated
  USING (
    EXISTS (
      SELECT 1 FROM profiles WHERE id = auth.uid() AND role = 'admin'
    )
  );
```

5. Members UPDATE own events if status is draft/pending_review/rejected:
```sql
CREATE POLICY "members_update_own_events" ON events
  FOR UPDATE TO authenticated
  USING (
    created_by = auth.uid()
    AND status IN ('draft', 'pending_review', 'rejected')
  )
  WITH CHECK (
    created_by = auth.uid()
  );
```

6. Moderators UPDATE status to published/rejected:
```sql
CREATE POLICY "moderators_update_event_status" ON events
  FOR UPDATE TO authenticated
  USING (
    EXISTS (
      SELECT 1 FROM profiles WHERE id = auth.uid() AND role IN ('moderator', 'admin')
    )
    AND status != 'deleted'
  )
  WITH CHECK (
    EXISTS (
      SELECT 1 FROM profiles WHERE id = auth.uid() AND role IN ('moderator', 'admin')
    )
  );
```

7. Admins full access on events:
```sql
CREATE POLICY "admins_all_events" ON events
  FOR ALL TO authenticated
  USING (
    EXISTS (
      SELECT 1 FROM profiles WHERE id = auth.uid() AND role = 'admin'
    )
  )
  WITH CHECK (
    EXISTS (
      SELECT 1 FROM profiles WHERE id = auth.uid() AND role = 'admin'
    )
  );
```

8. Event audit log — authenticated INSERT:
```sql
CREATE POLICY "authenticated_insert_audit_log" ON event_audit_log
  FOR INSERT TO authenticated
  WITH CHECK (true);
```

9. Event audit log — admin-only SELECT:
```sql
CREATE POLICY "admins_select_audit_log" ON event_audit_log
  FOR SELECT TO authenticated
  USING (
    EXISTS (
      SELECT 1 FROM profiles WHERE id = auth.uid() AND role = 'admin'
    )
  );
```

IMPORTANT: Check if events table already has RLS enabled and existing policies. If existing policies conflict, they need to be dropped first. The admin events page currently uses service role, so existing RLS may not exist on events. Check the original migration.
</action>
