## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline label padding |
| sm | 8px | Input icon gap, result row icon gap |
| md | 16px | Input horizontal padding, result row horizontal padding |
| lg | 24px | Overlay internal section padding |
| xl | 32px | Desktop: overlay top offset from viewport (pt-[15vh] equivalent) |
| 2xl | 48px | Empty state vertical padding |
| 3xl | 64px | — (not used in this phase) |

Exceptions:
- Result row avatars: fixed 32px (w-8 h-8) — compact, matches header avatar size (source: CONTEXT.md)
- Touch targets for close button (X): minimum 44px hit area (absolute positioned top-right on mobile)
- Overlay max-width: max-w-2xl (672px) on desktop (source: CONTEXT.md specifics)
- Results scroll area max-height: max-h-[60vh] on desktop, uncapped on mobile (fills screen above input)

## Component Anatomy

### Desktop Layout (≥ 640px breakpoint)

```
fixed inset-0 z-[9999]           ← Backdrop layer (bg-black/40 backdrop-blur-sm)
fixed inset-0 z-[10000]          ← Portal container (flex items-start justify-center pt-[10vh])
  motion.div                     ← Overlay panel (max-w-2xl w-full bg-white rounded-2xl shadow-2xl)
    ┌─────────────────────────────────────────────────────┐
    │  [Search icon]  [Input: "Search members, events…"]  [X] │  ← Input bar (px-4 py-3 border-b)
    ├─────────────────────────────────────────────────────┤
    │  [All] [Members] [Events] [Courses] [Pages]         │  ← Filter pills (px-4 py-2 border-b)
    ├─────────────────────────────────────────────────────┤
    │  MEMBERS                                            │  ← Category group header
    │  [avatar] Name          Subtitle      [msg] [map]  │  ← Result row
    │  [avatar] Name          Subtitle      [msg]        │  ← Result row (no address)
    │  EVENTS                                             │
    │  [icon]   Event title   Date          →             │  ← Result row
    ├─────────────────────────────────────────────────────┤
    │  ↑↓ navigate   ↵ open   Esc close         [Cmd K]  │  ← Keyboard hint bar (hidden mobile)
    └─────────────────────────────────────────────────────┘
```

### Mobile Layout (< 640px breakpoint)

```
fixed inset-0 z-[10000]          ← Full-screen overlay (bg-white flex flex-col)
  [X close button]               ← absolute top-3 right-4
  flex-col-reverse               ← Input at visual bottom, results scroll above
    ┌─────────────────────────────┐
    │  Results scroll area        │  ← flex-1 overflow-y-auto
    │  [result rows...]           │
    ├─────────────────────────────┤
    │  [All][Members][Events]...  │  ← Filter pills: overflow-x-auto, no wrap
    ├─────────────────────────────┤
    │  [🔍] Search...         [X] │  ← Input bar anchored at bottom
    └─────────────────────────────┘
    (keyboard hint bar: hidden on mobile)
```

Source: CONTEXT.md decisions (mobile layout, flex-col-reverse, keyboard hint hidden on mobile)

## Animation Contract

All animations use framer-motion AnimatePresence + motion.div. Source: CONTEXT.md + FlowPlayerModal pattern.

| Event | Animation | Duration | Easing |
|-------|-----------|----------|--------|
| Overlay enter | opacity 0→1, scale 0.95→1.0 | 200ms | easeOut |
| Overlay exit | opacity 1→0, scale 1.0→0.95 | 200ms | easeOut |
| Result row hover | bg transition | 150ms | default |
| Filter pill active | bg/text color swap | 100ms | default |

Z-index: backdrop at `z-[9999]`, panel container at `z-[10000]` — matches FlowPlayerModal pattern (source: CONTEXT.md Claude's Discretion, FlowPlayerModal.tsx)

## Interaction States

### Search Input
- On open: auto-focus (50ms timeout to allow DOM render — matches SearchModal.tsx pattern)
- Placeholder: "Search members, events, courses, pages…"
- Icon: Search (lucide-react) — 20px, `text-slate-400` idle, `text-primary` when query non-empty
- Clear input: X button appears inside input when query.length > 0 (16px icon, `text-slate-400 hover:text-slate-600`)

### Filter Pills (All / Members / Events / Courses / Pages)
- Container: `flex gap-2 px-4 py-2 border-b border-slate-100` on desktop; `flex gap-2 px-4 py-2 overflow-x-auto` on mobile (no wrap)
- Inactive pill: `px-3 py-1 rounded-full text-xs font-medium text-slate-600 bg-slate-100 hover:bg-slate-200 transition-colors`
- Active pill: `px-3 py-1 rounded-full text-xs font-medium text-white bg-primary`
- Default active pill: "All"

### Result Rows
- Container: `divide-y divide-slate-100`
- Default row: `flex items-center gap-3 px-4 py-3 hover:bg-slate-50 transition-colors cursor-pointer`
- Keyboard-highlighted row: `bg-primary/5` background + `border-l-2 border-primary` left accent
- Best/top result (first result overall): `border-l-2 border-primary bg-primary/5` — persistent, not only on hover (source: CONTEXT.md specifics, highlight color #345c83)
- Avatar: `w-8 h-8 rounded-full object-cover` — 32px, fallback initials with `bg-primary-100 text-primary-dark`
- Category icon (for non-member rows): 20px lucide icon in `text-slate-400`

### Member Row Contextual Action Icons
- Message icon: `MessageCircle` (lucide-react, 16px) — `text-slate-400 hover:text-primary transition-colors` — always shown for member rows
- Map/directions icon: `MapPin` (lucide-react, 16px) — `text-slate-400 hover:text-primary transition-colors` — shown only when `has_full_address === true` (source: REQUIREMENTS.md SRCH-06, CONTEXT.md specifics)
- Icons grouped right: `flex items-center gap-1 ml-auto`

### Keyboard Navigation
- ArrowDown: move highlight to next result (wraps to first when at last)
- ArrowUp: move highlight to previous result (wraps to last when at first)
- Enter: navigate to highlighted result's href (router.push), close overlay
- Escape: close overlay
- Source: REQUIREMENTS.md SRCH-04

### Close Triggers
- Esc key
- X button click
- Backdrop click (outside overlay panel)
- Source: REQUIREMENTS.md SRCH-02

## States: Empty, Loading, Error

### Empty — No query typed (initial open state)
```
Heading: "Search GOYA"
Body:    "Find members, events, courses, and pages."
Visual:  Search icon (40px, text-slate-300), centered in results area
```

### Empty — Query too short (< 2 chars)
```
Body: "Keep typing…"
Visual: centered text-sm text-slate-400, no icon
```

### Empty — Query returned no results
```
Heading: "No results for "{query}""
Body:    "Try a different search term."
Visual:  centered, text-sm, text-slate-400
```

### Loading (debounce window / fetch in progress)
- Show 3 skeleton rows in results area
- Skeleton pill widths: first 200px, second 140px, third 180px (animate-pulse, bg-slate-100, rounded-md, h-4)
- Filter pills remain interactive during loading

### Error (fetch failed)
```
Body: "Search unavailable. Please try again."
Visual: centered text-sm text-red-500
```

Source: REQUIREMENTS.md INTG-03 + CONTEXT.md decisions (loading skeleton during fetch)

## Accessibility Contract

- `role="dialog"` on overlay panel + `aria-modal="true"` + `aria-label="Search"`
- `role="listbox"` on results container
- `role="option"` + `aria-selected` on each result row (true when keyboard-highlighted)
- Focus trap: Tab/Shift+Tab cycles through input → filter pills → result rows → X button
- On overlay close: return focus to the element that opened the overlay (search icon in header)
- Filter pills: `role="radio"` group or `role="tab"` pattern with `aria-pressed` on each pill
- Skeleton rows: `aria-busy="true"` on results container during loading

## React Architecture Contract

- Render via `createPortal(…, document.body)` — matches FlowPlayerModal (source: CONTEXT.md)
- State managed via `SearchContext` (React Context) — one instance, mounted at layout level in ClientProviders (source: CONTEXT.md + STATE.md Research Notes)
- `useCallback + useRef` debounce pattern, 200ms — no extra library (source: STATE.md Research Notes)
- Result cache: `useRef<Map<string, SearchResult[]>>()` keyed by trimmed lowercase query (source: STATE.md Research Notes)
- Keyboard shortcut (Cmd+K / Ctrl+K): handled in Phase 53 (INTG-01 is out of scope for Phase 51)
- Adapt debounce, keyboard nav, scoring structure from `app/components/docs/SearchModal.tsx`

Phase 51 delivers the UI component with mock/stub data. Real API wired in Phase 52.
