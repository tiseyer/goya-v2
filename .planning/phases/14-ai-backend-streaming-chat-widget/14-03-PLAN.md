    **Export async function getOrCreateSession(params: { userId?: string, anonymousId?: string, existingSessionId?: string }): Promise<{ session_id: string, messages: Array<{ id: string, role: string, content: string, created_at: string }> }>**
    1. If existingSessionId provided: verify it exists in `chat_sessions` and belongs to the user/anonymous. If valid, load and return messages.
    2. If no existing session: INSERT new row into `chat_sessions` with user_id or anonymous_id. Return empty messages array.
    3. Messages query: SELECT from `chat_messages` WHERE `session_id`, ORDER BY `created_at ASC`, LIMIT 100.
    4. Return `{ session_id, messages }`.

    **Export async function getChatHistory(sessionId: string): Promise<Array<{ id: string, role: string, content: string, created_at: string }>>**
    - SELECT from `chat_messages` WHERE `session_id`, ORDER BY `created_at ASC`, LIMIT 100.
    - Return array of messages.

    **Export async function deleteSession(sessionId: string): Promise<void>**
    - DELETE FROM `chat_messages` WHERE `session_id`
    - DELETE FROM `chat_sessions` WHERE `id = sessionId`
    - Both via service client.

    **Export async function getAnonymousId(): Promise<string>**
    - Import `cookies` from `next/headers`
    - Check for existing `goya_chat_session` cookie
    - If exists, return its value
    - If not, generate UUID via `crypto.randomUUID()`, set cookie with `cookies().set('goya_chat_session', id, { httpOnly: true, secure: true, sameSite: 'lax', maxAge: 30 * 24 * 60 * 60, path: '/' })`, return id
    - This is the guest session persistence mechanism per CONTEXT.md

    **Export async function getCurrentUserId(): Promise<string | null>**
    - Import `createSupabaseServerClient` from `@/lib/supabaseServer`
    - Call `supabase.auth.getUser()`
    - Return `user?.id ?? null`

    **Replace mock send handler with real streaming:**
    ```
    async function handleSend(userMessage: string):
    1. Add user message to local messages state immediately (optimistic)
    2. Set isTyping = true
    3. Fetch POST /api/chatbot/message with body:
       { session_id: sessionId, message: userMessage, anonymous_id: anonymousId }
    4. If response.status === 429:
       - Set isRateLimited = true
       - Add rate-limit bubble to messages with content from CONTEXT: "You've reached the message limit for this hour. Please check back soon."
       - Set isTyping = false
       - Return
    5. If response not ok:
       - Add error assistant bubble "Something went wrong. Please try again."
       - Set isTyping = false
       - Return
    6. Read stream via response.body.getReader() + TextDecoder:
       - For each JSON line parsed:
         - type "token": first token sets isTyping=false, isStreaming=true, creates assistant message entry. Subsequent tokens append to content of last assistant message.
         - type "done": set isStreaming=false. Update sessionId from response if different.
         - type "escalation": set isEscalated=true, isTyping=false. Add escalation bubble with the message content. Store sessionId.
         - type "error": set isTyping=false, add error bubble.
    7. In finally block: ensure isTyping=false, isStreaming=false
    ```

    **Parse streaming lines carefully:** The stream sends JSON-per-line (`\n` delimited). Buffer partial lines (a chunk may split a JSON object across two reads). Pattern:
    ```
    let buffer = ''
    // in read loop:
    buffer += decoder.decode(chunk, { stream: true })
    const lines = buffer.split('\n')
    buffer = lines.pop() ?? ''  // keep incomplete last line in buffer
    for (const line of lines) {
      if (!line.trim()) continue
      const data = JSON.parse(line)
      // handle data.type
    }
    ```
