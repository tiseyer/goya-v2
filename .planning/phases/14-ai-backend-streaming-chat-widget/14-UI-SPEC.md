# Phase 14 — UI Design Contract

> Visual and interaction contract for the floating chat widget. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> This phase introduces a new surface type — a floating overlay widget — distinct from all admin pages.

| Property | Value |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Message bubble inner padding (vertical), avatar gap |
| md | 16px | Message bubble inner padding (horizontal), input area padding |
| lg | 24px | Widget header padding, section padding |
| xl | 32px | Widget panel bottom offset from viewport edge |
| 2xl | 48px | Floating button bottom offset from viewport edge |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Floating button: 56x56px (touch target minimum 44px met, 56px provides visual weight)
- Mattea avatar in header: 36x36px
- Mattea avatar in message thread: 28x28px
- Widget panel: exactly 380px wide, 560px tall on desktop (from CONTEXT.md CHAT-02)
- Mobile fullscreen: 100dvh height, 100vw width

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body (message text) | 14px (text-sm) | 400 (regular) | 1.5 |
| Label (header name, input placeholder) | 14px (text-sm) | 600 (semibold) | 1.4 |
| Caption (timestamp, online indicator) | 12px (text-xs) | 400 (regular) | 1.4 |
| Greeting / system message | 14px (text-sm) | 400 (regular) | 1.6 |

Note: Only 2 font sizes (12px, 14px) and 2 weights (400, 600) used in the widget. Heading size is not needed — this is a compact overlay, not a page.

Source: matches existing ConfigurationTab pattern (text-sm / text-xs, font-semibold for labels)

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | var(--goya-surface) = #ffffff light / #1A1D27 dark | Widget panel background, message list area |
| Secondary (30%) | var(--background-secondary) = #f8f9fa light / #232733 dark | User message bubble background, input field background, header background |
| Accent (10%) | var(--goya-primary) = #345c83 light / #5B9ABF dark | Floating button, send button, streaming cursor, online indicator dot |
| Destructive | var(--goya-accent) = #831618 light / #DC2626 dark | Delete chat history button/icon only |

Accent reserved for:
1. Floating chat button background
2. Send message button background
3. Streaming cursor animation (blinking bar appended to in-progress token)
4. Online indicator dot in widget header
5. Escalation message border-left accent stripe

Secondary (user bubble): `var(--background-secondary)` with `var(--goya-border)` border — visually distinct from Mattea bubbles which use `var(--goya-surface)` with `var(--goya-border)` border.

Mattea bubble: white surface with subtle border. User bubble: secondary surface with subtle border. This gives a clear left/right visual distinction without using accent color for bubbles.

Source: globals.css token audit. Dark mode variants sourced from .dark {} block.

## Component Inventory

### ChatWidget (root)
- Mounted in app/layout.tsx, conditionally rendered when pathname does NOT start with /admin
- Reads is_active from GET /api/chatbot/config on mount — renders nothing if false (CHAT-12)
- Contains: FloatingButton + ChatPanel (panel hidden until button clicked)

### FloatingButton
- 56x56px circle, fixed bottom-right: `bottom-6 right-6` (24px = lg token)
- Background: `bg-primary` with `shadow-elevated`
- Icon: MessageCircle from lucide-react, 24px, white
- On click: opens ChatPanel with slide-up animation
- aria-label: "Open chat with Mattea"
- Hidden on /admin/* routes

### ChatPanel (desktop)
- Fixed `bottom-24 right-6` (above the floating button, 96px offset = 6rem)
- Dimensions: 380px wide, 560px tall
- Border: 1px solid `var(--goya-border)`, border-radius: 16px
- Background: `var(--goya-surface)`
- Shadow: `shadow-elevated`
- Animation: `slideUp` keyframe (already defined in globals.css) on open; `opacity 0 → 1` on close
- z-index: 50

### ChatPanel (mobile, breakpoint < 768px)
- Position: fixed inset-0 (full screen)
- Border-radius: 0
- Height: 100dvh

### ChatHeader
- Height: 56px, `px-4` (16px), flex row items-center
- Border-bottom: 1px solid `var(--goya-border)`
- Background: `var(--background-secondary)`, border-radius: 16px 16px 0 0
- Left: Mattea avatar 36x36px circle + name label (semibold 14px) + online indicator dot (8px, bg-primary)
- Right: two icon buttons (ghost variant, 32x32px)
  - RotateCcw icon (16px) — "New Chat" — aria-label: "Start new chat"
  - Trash2 icon (16px) — "Delete chat history" — aria-label: "Delete chat history", accent-colored (destructive)
- Close button: X icon (16px) — `ml-auto` — aria-label: "Close chat"

### MessageList
- Flex-col, scrollable overflow-y-auto, padding: `px-4 py-4`
- `scrollbar-none` class (already defined in globals.css)
- Auto-scrolls to bottom on new message
- Contains: GreetingBubble (on fresh session), MessageBubble[] items

### GreetingBubble
- Mattea left-aligned bubble
- Copy (from CONTEXT.md CHAT-06): "Namaste! I'm Mattea, your GOYA guide. How can I help you today?"
- Rendered as first item in empty/new conversation

### MessageBubble (user)
- Right-aligned: `ml-auto max-w-[75%]`
- Background: `var(--background-secondary)`, border: 1px `var(--goya-border)`, border-radius: 12px 12px 2px 12px
- Padding: `px-3.5 py-2.5` (14px / 10px)
- Text: 14px regular, `var(--foreground)`

### MessageBubble (Mattea / assistant)
- Left-aligned: `mr-auto max-w-[75%]`, flex row with avatar
- Avatar: 28x28px circle, `shrink-0 mt-0.5`
- Bubble: Background `var(--goya-surface)`, border: 1px `var(--goya-border)`, border-radius: 2px 12px 12px 12px
- Padding: `px-3.5 py-2.5`
- Text: 14px regular, `var(--foreground)`
- Streaming state: append blinking cursor `|` in `text-primary` after last character while tokens are incoming

### TypingIndicator
- Shown when AI call has started but no first token received yet
- Same layout as Mattea bubble (left-aligned with avatar)
- Content: three animated dots (pulse stagger), `bg-foreground-tertiary`, 6px diameter each, `gap-1`
- aria-label: "Mattea is typing"

### EscalationBubble
- Left-aligned Mattea bubble with a 3px left border in `var(--goya-primary)`
- Background: `var(--goya-primary-50)` (very subtle tint)
- Copy (from CONTEXT.md): "That's a great question — I'll check with our team and get back to you, usually within 48 hours."
- Shown in place of a normal assistant bubble when escalation flag returned from API

### RateLimitBubble
- Left-aligned Mattea bubble
- Background: `var(--goya-accent-50)`, border: 1px `var(--goya-accent)/20`
- Copy: "You've reached the message limit for this hour. Please check back soon."

### ChatInput
- Bottom section: height 60px, `px-4 py-3`, border-top: 1px `var(--goya-border)`
- Background: `var(--background-secondary)`
- Textarea (single-line, auto-expand max 3 rows): `bg-background-tertiary`, border: 1px `var(--goya-border)`, `rounded-xl px-3 py-2 text-sm`
- Placeholder: "Ask Mattea anything..."
- Send button: 36x36px circle, `bg-primary`, ArrowUp icon 16px white, `rounded-full`
- Send button disabled (empty input or streaming): `opacity-50 cursor-not-allowed`
- Submit on Enter (no Shift): sends message. Shift+Enter: newline.

## Interaction States

| State | Visual |
|-------|--------|
| Widget closed | FloatingButton visible, panel hidden |
| Widget open, no history | GreetingBubble shown |
| User sends message | User bubble appends immediately, TypingIndicator appears |
| First token arrives | TypingIndicator replaced by streaming Mattea bubble |
| Streaming | Cursor `\|` blinks at end of partial text |
| Stream complete | Cursor removed |
| Escalation triggered | EscalationBubble shown, input disabled |
| Rate limit hit | RateLimitBubble shown, input disabled |
| Network error | Error bubble (see Copywriting Contract) |
| Delete confirmation | Inline confirm row below trash button: "Delete all?" + "Yes, delete" (danger) + "Cancel" (ghost) |
| New chat | Clears message list, shows GreetingBubble, new session_id generated |

## Animation Contract

All animations use existing globals.css keyframes or Tailwind transitions:

| Element | Animation | Duration |
|---------|-----------|----------|
| Panel open | slideUp (translate Y 100% → 0) | 280ms ease-out |
| Panel close | opacity 1 → 0, translate Y 0 → 8px | 180ms ease-in |
| Bubble appear | stepIn (opacity 0 + translateY 8px → normal) | 220ms ease |
| TypingIndicator dots | CSS pulse stagger (dot 1: 0ms, dot 2: 150ms, dot 3: 300ms delay) | 1200ms infinite |
| Streaming cursor blink | opacity 1 → 0 cycle | 600ms step |

Source: globals.css already defines `slideUp`, `stepIn`, `fadeIn` keyframes. Reuse them — do not add new keyframe definitions.

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Focus trap | Focus trapped inside ChatPanel when open; Escape closes panel |
| Focus return | Focus returns to FloatingButton on panel close |
| Keyboard send | Enter submits, Shift+Enter newline |
| Live region | MessageList has `aria-live="polite"` for streaming tokens |
| Typing indicator | `aria-label="Mattea is typing"` on TypingIndicator |
| Role | ChatPanel root: `role="dialog" aria-label="Chat with Mattea" aria-modal="true"` |
| Scroll | MessageList uses `scrollbar-none` but remains keyboard-scrollable |
| Touch target | FloatingButton 56x56px — exceeds 44px minimum |
| Contrast | All body text on surface backgrounds passes WCAG AA at 4.5:1 minimum |
