    - Consecutive same-question detection: if the last 2 user messages in recentMessages are substantially the same as the current message (normalized: lowercase, trimmed, >80% character match using simple ratio), return true

    **Step 5: Create lib/chatbot/chat-service.ts:**
    - Import `decrypt` from `@/lib/secrets/encryption`
    - Import `getSupabaseService` from `@/lib/supabase/service`
    - Import OpenAI from `openai` and Anthropic from `@anthropic-ai/sdk`
    - Import `detectEscalation` from `./escalation`
    - Import types from `./types`

    Export async function `streamChatResponse(params: { sessionId: string | null, message: string, userId: string | null, anonymousId: string | null }): Promise<ChatStreamResult>`:

    1. **Load config**: Query `chatbot_config` table via service client, get single row. If `!is_active`, throw error "Chatbot is not active".
    2. **Resolve or create session**: If sessionId provided, verify it exists in `chat_sessions` and belongs to user/anonymous. If not provided, INSERT new session with user_id or anonymous_id, return new session_id.
    3. **Load conversation history**: SELECT last 20 messages from `chat_messages` WHERE session_id, ordered by created_at ASC. Map to `{ role, content }` array.
    4. **Check escalation**: Call `detectEscalation(message, history)`. If true:
       - INSERT into `support_tickets` with `{ session_id, user_id, question: message, status: 'open' }`
       - UPDATE `chat_sessions` SET `is_escalated = true`
       - Save user message to `chat_messages`
       - Save escalation response to `chat_messages` with role 'assistant' and content "That's a great question -- I'll check with our team and get back to you, usually within 48 hours."
       - Return `{ stream: new ReadableStream that immediately closes after writing the escalation JSON, session_id, escalated: true }`
    5. **Decrypt provider key**: Query `admin_secrets` WHERE `id = config.selected_key_id`, decrypt using `decrypt(encrypted_value, iv)`. Extract provider and model from the secret's `provider` and `model` columns.
    6. **Build FAQ context**: Query `faq_items` WHERE `status = 'published'`, format as:
       ```
       <faq_context>
       <item><q>Question here</q><a>Answer here</a></item>
       ...
       </faq_context>
       ```
    7. **Build messages array**: system prompt (config.system_prompt + FAQ context block), then conversation history, then new user message.
    8. **Save user message** to `chat_messages` immediately (before AI call): INSERT `{ session_id, role: 'user', content: message }`.
    9. **Call AI provider**:
       - If provider === 'openai': use `new OpenAI({ apiKey }).chat.completions.create({ model, messages, stream: true })`. Convert the OpenAI stream to a ReadableStream that writes JSON chunks: `{ type: 'token', content: string }` per token, then `{ type: 'done' }` at end. In the onFinish equivalent (after stream completes), save the full assembled assistant message to `chat_messages`.
       - If provider === 'anthropic': use `new Anthropic({ apiKey }).messages.stream({ model, max_tokens: 1024, system: systemPrompt, messages: historyWithoutSystem })`. Convert similarly. Save full message on completion.
    10. **Return** `{ stream, session_id }`.

    Use a TransformStream to convert SDK-specific streaming into a uniform JSON-line format. Each line is a JSON object terminated by `\n`. Format:
    - Token: `{"type":"token","content":"word "}\n`
    - Done: `{"type":"done","session_id":"uuid"}\n`
    - Escalation: `{"type":"escalation","message":"That's a great question...","session_id":"uuid"}\n`
    - Error: `{"type":"error","message":"Something went wrong"}\n`

    IMPORTANT: The full assistant response text must be accumulated during streaming and saved to chat_messages AFTER the stream completes (use a finally block or stream controller close handler). Do NOT save partial responses.
