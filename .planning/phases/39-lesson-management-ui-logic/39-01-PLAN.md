**app/admin/courses/lesson-actions.ts** — Create 'use server' file with these actions:

1. `fetchLessons(courseId: string)` — SELECT * FROM lessons WHERE course_id = courseId ORDER BY sort_order ASC. Use `createSupabaseServerActionClient`. Return `{ data: Lesson[], error: string | null }`.

2. `createLesson(courseId: string, formData: LessonFormData)` — Calculate sort_order as: query MAX(sort_order) for this course, new sort_order = max + 1024 (or 1024 if no lessons). INSERT into lessons table with course_id, all formData fields, and computed sort_order. Log audit action via `logAdminCourseAction(courseId, 'edited', { added_lesson: formData.title })`. Return `{ data: Lesson | null, error: string | null }`.

3. `updateLesson(lessonId: string, courseId: string, formData: LessonFormData)` — UPDATE lessons SET all formData fields, updated_at = new Date().toISOString() WHERE id = lessonId. Log audit. Return `{ data: Lesson | null, error: string | null }`.

4. `deleteLesson(lessonId: string, courseId: string, lessonTitle: string)` — DELETE FROM lessons WHERE id = lessonId. Log audit action `logAdminCourseAction(courseId, 'edited', { deleted_lesson: lessonTitle })`. Return `{ error: string | null }`.

5. `reorderLesson(lessonId: string, newSortOrder: number)` — UPDATE lessons SET sort_order = newSortOrder, updated_at = new Date().toISOString() WHERE id = lessonId. This is a single-row write for float midpoint reorder. Return `{ error: string | null }`.

Use `createSupabaseServerActionClient` for all actions (same pattern as category-actions.ts). Do NOT use getSupabaseService — let RLS handle permissions.

Use `useSortable({ id: lesson.id })` with `CSS.Transform.toString(transform)` and `transition` style, plus `isDragging ? 0.5 : 1` opacity — identical to ProductsTable pattern.

**handleDragEnd:**
1. Get active and over from event. If !over or same id, return.
2. Find oldIndex and newIndex in lessons array.
3. Call `arrayMove(lessons, oldIndex, newIndex)` to get newOrder, update local state.
4. Calculate float midpoint sort_order:
   - If newIndex === 0: `newOrder[1]?.sort_order ? newOrder[1].sort_order / 2 : 1024`
   - If newIndex === newOrder.length - 1: `newOrder[newOrder.length - 2].sort_order + 1024`
   - Else: `(newOrder[newIndex - 1].sort_order + newOrder[newIndex + 1].sort_order) / 2`
5. Call `reorderLesson(activeId, newSortOrder)` — single row write.
6. Update local state sort_order for the moved item.
