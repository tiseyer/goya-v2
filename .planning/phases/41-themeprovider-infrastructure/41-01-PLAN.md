```typescript
async function getAnalyticsSettings(): Promise<Record<string, string> | null> {
  const supabaseUrl = process.env.NEXT_PUBLIC_SUPABASE_URL;
  const supabaseKey = process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY;
  const url = `${supabaseUrl}/rest/v1/site_settings?key=in.(...)&select=key,value`;
  const res = await fetch(url, {
    headers: { apikey: supabaseKey, Authorization: `Bearer ${supabaseKey}` },
    cache: 'no-store',
  });
  const rows = await res.json() as Array<{ key: string; value: string }>;
}
```

Create app/components/ThemeColorProvider.tsx as a **React Server Component** (no 'use client' directive). This component:

1. Fetches color settings from site_settings using the SAME fetch pattern as getAnalyticsSettings in layout.tsx (direct REST API call with anon key, cache: 'no-store'). Query for keys: `brand_colors`, `role_colors`, `maintenance_indicator_color`.

2. Parses the JSON values:
   - `brand_colors` value is a JSON string matching BrandColors interface (e.g., `{"primary":"#345c83","accent":"#831618",...}`)
   - `role_colors` value is a JSON string matching RoleColors interface
   - `maintenance_indicator_color` value is a plain hex string (e.g., `"#F59E0B"`)

3. Merges fetched values with defaults (spread defaults first, then DB values on top). This ensures partial saves still work — if admin only changed primary, the rest stay at defaults.

4. Renders a `<style>` tag that sets CSS custom properties on `:root`:
   ```
   :root {
     --color-primary: #345c83;
     --color-accent: #831618;
     --color-bg: #ffffff;
     --color-surface: #ffffff;
     --color-border: #e2e8f0;
     --color-foreground: #1e293b;
     --color-student: #6366f1;
     --color-teacher: #345c83;
     --color-wellness: #059669;
     --color-school: #d97706;
     --color-moderator: #7c3aed;
     --color-admin: #dc2626;
     --color-maintenance: #F59E0B;
   }
   ```

Important implementation details:
- Wrap the entire fetch+parse in try/catch. On ANY error, fall back to defaults silently (no broken pages if DB is down).
- The component returns ONLY a `<style>` JSX element (or null if using defaults and you want to skip — but always rendering is simpler and ensures consistency).
- Do NOT use getSupabaseService() — layout.tsx runs at the edge and needs the lightweight REST fetch pattern. Follow getAnalyticsSettings exactly.
- The `<style>` tag should use `dangerouslySetInnerHTML` to inject the CSS string.

- Place `<ThemeColorProvider />` INSIDE the `<html>` tag, BEFORE `<body>`. This ensures CSS variables are set before any body content renders.

The updated layout structure should be:

Do NOT move or change getAnalyticsSettings, ClientProviders, or any other existing layout code. Only add the import and the single component placement.
