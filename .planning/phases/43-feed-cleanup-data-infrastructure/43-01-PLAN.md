Create `lib/dashboard/profileCompletion.ts` as a pure computation module (no DB queries, no server-only needed).

**1. Export `isFieldComplete(value: unknown): boolean`**
Must handle these cases per the JSONB pitfall:
```typescript
export function isFieldComplete(value: unknown): boolean {
  if (value === null || value === undefined) return false
  if (typeof value === 'string') return value.trim().length > 0
  if (Array.isArray(value)) return value.length > 0
  return Boolean(value)
}
```

**2. Export `WEIGHTED_FIELDS` constant array**
Six fields with weights summing to 100. Each has key (matching profiles column), weight, label (for UI), and href (deep link to settings section):
```typescript
export const WEIGHTED_FIELDS = [
  { key: 'avatar_url',      weight: 20, label: 'Profile photo',   href: '/settings' },
  { key: 'bio',             weight: 20, label: 'Bio',             href: '/settings' },
  { key: 'location',        weight: 15, label: 'Location',        href: '/settings' },
  { key: 'teaching_styles', weight: 15, label: 'Teaching styles',  href: '/settings' },
  { key: 'website',         weight: 10, label: 'Website or social', href: '/settings' },
  { key: 'avatar_url' field covers profile photo (20%), bio covers text bio (20%), etc.
] as const
```

Wait — per CONTEXT.md the weights are: avatar 20%, bio 20%, location 15%, teaching_styles/practice_types 15%, website/social 10%, event/course 20%. The last item (event/course 20%) is tricky — it checks whether the user has created at least one event OR course. This requires data not on the profile object.

Handle this by making getProfileCompletion accept an optional `hasContent` boolean parameter that the caller computes:

```typescript
export const WEIGHTED_FIELDS = [
  { key: 'avatar_url',      weight: 20, label: 'Profile photo',      href: '/settings' },
  { key: 'bio',             weight: 20, label: 'Bio',                href: '/settings' },
  { key: 'location',        weight: 15, label: 'Location',           href: '/settings' },
  { key: 'teaching_styles', weight: 15, label: 'Teaching styles',    href: '/settings' },
  { key: 'website',         weight: 10, label: 'Website or social',  href: '/settings' },
] as const

export const CONTENT_FIELD = { key: 'has_content', weight: 20, label: 'Share an event or course', href: '/events' } as const
```

**3. Export `getProfileCompletion(profile: Record<string, unknown>, hasContent?: boolean)`**
Returns `{ score: number, missing: Array<{ key: string, label: string, href: string }>, complete: string[] }`.

Logic:
- For each of the 5 WEIGHTED_FIELDS, use isFieldComplete(profile[field.key]) to check.
- For the CONTENT_FIELD, use the hasContent boolean (defaults to false).
- Score = sum of weights for complete fields.
- Missing = array of { key, label, href } for incomplete fields.
- Complete = array of keys for complete fields.

For the website/social field (weight 10), check if ANY of website, instagram, or youtube is complete (social_fields = ['website', 'instagram', 'youtube']). If at least one is filled, that field scores as complete.

**4. Export TypeScript types**
```typescript
export type ProfileCompletionResult = {
  score: number
  missing: Array<{ key: string; label: string; href: string }>
  complete: string[]
}
```
