Review the CourseForm for mobile responsiveness issues per ACF-07. Check and fix:

1. **Grid layouts** — All `grid` containers must use `grid-cols-1 sm:grid-cols-2` pattern (single column on mobile, two on desktop). Never use `grid-cols-2` without a `grid-cols-1` mobile fallback.

2. **Card sections** — Verify each card section `<div className="border border-border rounded-xl p-6 ...">` uses responsive padding: `p-4 sm:p-6` so cards don't eat too much space on small screens.

3. **Gradient picker row** — The "From" and "To" color pickers with hex labels must wrap on mobile. Use `flex flex-wrap` and ensure the hex mono text doesn't overflow. Verify the gradient preview bar is `w-full`.

4. **Duration slider** — Ensure the `<input type="range">` has `w-full` and the label + formatted value don't overlap on narrow screens. The slider thumb must be large enough for touch: add `[&::-webkit-slider-thumb]:w-5 [&::-webkit-slider-thumb]:h-5` if needed.

5. **Select inputs** — All selects must have `w-full` and not clip option text. Category color dot indicator must not break layout on small screens.

6. **Textarea** — Both description textareas should have `min-h-[80px]` on mobile.

7. **Submit/Cancel actions** — The button row should use `flex flex-col sm:flex-row` so buttons stack on very narrow screens, or keep `flex-row` if buttons fit. Either way, ensure no horizontal overflow.

8. **Smooth transitions** — Add `transition-all duration-200` to card sections for any interactive state changes.

9. **Form max width** — Keep `max-w-3xl` but ensure it doesn't cause issues within the admin layout. It should be `max-w-3xl mx-auto` or just `max-w-3xl` depending on the parent container.

Apply fixes as needed. If the form from Plan 01 already handles all these correctly, note that — but still verify each point.
