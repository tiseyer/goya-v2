**Migration 3 — 20260358_chat_sessions_messages.sql** (per INFRA-04):
```sql
CREATE TABLE chat_sessions (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id uuid REFERENCES profiles(id) ON DELETE SET NULL,
  anonymous_id text,
  created_at timestamptz NOT NULL DEFAULT now(),
  last_message_at timestamptz NOT NULL DEFAULT now(),
  is_escalated boolean NOT NULL DEFAULT false,
  expires_at timestamptz
);

ALTER TABLE chat_sessions ENABLE ROW LEVEL SECURITY;

-- Session owner can read their own sessions
CREATE POLICY "Users can read own sessions"
  ON chat_sessions FOR SELECT
  USING (auth.uid() = user_id);

-- Service role handles anonymous sessions and admin reads

CREATE TABLE chat_messages (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  session_id uuid NOT NULL REFERENCES chat_sessions(id) ON DELETE CASCADE,
  role text NOT NULL CHECK (role IN ('user', 'assistant')),
  content text NOT NULL,
  created_at timestamptz NOT NULL DEFAULT now()
);

ALTER TABLE chat_messages ENABLE ROW LEVEL SECURITY;

-- Users can read messages from their own sessions
CREATE POLICY "Users can read own session messages"
  ON chat_messages FOR SELECT
  USING (
    EXISTS (
      SELECT 1 FROM chat_sessions
      WHERE chat_sessions.id = chat_messages.session_id
        AND chat_sessions.user_id = auth.uid()
    )
  );
```

Add index on `chat_messages.session_id` and `chat_sessions.user_id` for query performance.

**app/admin/chatbot/chatbot-actions.ts** — Follow the exact pattern from `secrets-actions.ts`:
- `'use server'` + `import 'server-only'` at top
- Use `getSupabaseService()` for all DB operations (admin-only tables)
- Use `createSupabaseServerActionClient()` to get current user ID for created_by
- Return `{ success: true, ... } | { success: false, error: string }` pattern
- Call `revalidatePath('/admin/chatbot')` after mutations

Implement these server actions:

1. **getChatbotConfig()** — Fetch the single chatbot_config row via `.select('*').single()`. Return typed `ChatbotConfig`.

2. **saveChatbotConfig(data: { name, is_active, system_prompt, selected_key_id, guest_retention_days, avatar_url? })** — Upsert the config row. Validate: name required + max 100 chars, system_prompt required, guest_retention_days 1-365. If avatar_url is a File/Blob path, skip here (avatar upload handled separately in the UI component). **ADMIN-04 auto-fetch:** On first save, if `avatar_url` is null/empty in the existing config, automatically fetch the default Mattea image from `https://globalonlineyogaassociation.org/wp-content/uploads/2026/03/mattea.jpg`, upload it to `chatbot-avatars` bucket, and set `avatar_url` to the public URL. This is a one-time operation — subsequent saves skip if `avatar_url` is already set.

3. **uploadChatbotAvatar(formData: FormData)** — Extract file from FormData, validate JPG/PNG/WebP and max 2MB. Ensure Supabase storage bucket `chatbot-avatars` exists (create via `supabase.storage.createBucket('chatbot-avatars', { public: true })` if not found — idempotent). Upload to path `avatar-{timestamp}.{ext}`. Get public URL. Update chatbot_config.avatar_url. Return `{ success: true, url: string }`.

4. **listFaqItems()** — Select all faq_items ordered by created_at DESC. Join profiles table for creator_name: `.select('*, profiles!created_by(full_name)')`. Map the join result to flat FaqItem with creator_name.

5. **createFaqItem(question: string, answer: string)** — Validate both required, get current user ID for created_by, insert with status 'draft'. Return the new FaqItem.

6. **updateFaqItem(id: string, question: string, answer: string)** — Validate UUID format, both fields required, update by id.

7. **deleteFaqItem(id: string)** — Validate UUID format, delete by id.

8. **toggleFaqStatus(id: string, newStatus: FaqStatus)** — Validate UUID and status value, update only the status column.

Each action: wrap in try/catch, return `{ success: false, error }` on failure.
