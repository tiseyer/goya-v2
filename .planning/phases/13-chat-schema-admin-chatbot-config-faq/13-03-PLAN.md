Component accepts `{ initialItems: FaqItem[] }` — server-fetched list passed from page.tsx.

**Toolbar** — flex row with search input left, "Add FAQ" button right. Search input with `placeholder="Search FAQ entries..."`, `w-full max-w-sm` with magnifying glass icon from lucide-react (Search icon). Client-side filter: `items.filter(i => i.question.toLowerCase().includes(q) || i.answer.toLowerCase().includes(q))`. "Add FAQ" button: `bg-[#4e87a0] hover:bg-[#3d6f85] text-white text-sm font-semibold px-4 py-2 rounded-lg` (primary CTA uses bg-primary-light per UI-SPEC — NOT teal).

**Table** — wrapped in `overflow-x-auto`. `table className="w-full min-w-[640px]"`. Header row: `bg-slate-50 text-xs font-semibold uppercase tracking-widest text-[#6B7280]`. Columns: Question | Answer | Status | Created | By | Actions. Column widths: Question 30%, Answer 25%, Status auto, Created auto, By auto, Actions auto.

**Table body** — map filtered items to FaqRow components. Pass `expandedId` and `onExpand` callback (set expandedId or null).

**Empty state** — when items.length === 0 and search is empty: centered with clipboard/document icon from lucide-react, heading "No FAQ entries yet" (`text-base font-semibold text-[#1B3A5C] mt-4`), body "Add your first FAQ to help the chatbot answer common questions." (`text-sm text-[#6B7280] mt-2`), "Add FAQ" button below.

**Search no results** — when items filtered to 0 but items.length > 0: `text-center text-sm text-[#6B7280] py-12` with "No FAQ entries match your search."

**Add handler** — when modal returns new item, prepend to items array (appears at top).



**app/admin/chatbot/FaqRow.tsx** — `'use client'` component (per ADMIN-06).

Component accepts `{ item: FaqItem; isExpanded: boolean; onExpand: (id: string | null) => void; onDelete: (id: string) => void; onUpdate: (id: string, item: FaqItem) => void }`.

**Collapsed state** — standard table row. Question: truncated to ~80 chars with ellipsis (`line-clamp-1`). Answer: truncated to ~60 chars. Status: clickable badge — Published (`bg-emerald-50 text-emerald-700`) or Draft (`bg-yellow-50 text-yellow-700`). On click: optimistic toggle via `toggleFaqStatus`. During save: badge opacity-60. On failure: revert + toast "Failed to update status." Created: format as "Mar 27, 2026" using `new Date(item.created_at).toLocaleDateString('en-US', { month: 'short', day: 'numeric', year: 'numeric' })`. By: `item.creator_name ?? 'System'`. Actions: "Edit" text button (teal text), "Delete" text button (red text).

Row hover: `hover:bg-slate-50`.

**Expanded state** (when isExpanded=true) — row + expand area below. Expand area: `bg-slate-50 border-t border-slate-200 px-4 py-4`. Full-width Question textarea (label "Question"). Full-width Answer textarea (label "Answer"). Button row: "Save Changes" (primary, `bg-[#4e87a0]`) + "Discard Changes" (text button, `text-slate-500`). Save calls `updateFaqItem(id, question, answer)` — on success: update parent items state via onUpdate callback, collapse row. Discard: reset local textarea state to original, collapse.

**Delete flow** — inline confirmation within the row (NOT a modal). Click "Delete" replaces action buttons with: "Sure?" label + red "Delete" button + "Keep Entry" text button. During deletion: row fades to 50% opacity, buttons disabled. On success: call onDelete(id) to remove from parent state. "Keep Entry" reverts to normal action buttons.



**app/admin/chatbot/FaqModal.tsx** — `'use client'` component.

Component accepts `{ open: boolean; onClose: () => void; onCreated: (item: FaqItem) => void }`.

**Modal overlay** — `fixed inset-0 z-50 flex items-center justify-center` with `backdrop-blur-sm bg-black/30`.

**Modal card** — `max-w-md w-full mx-4 bg-white rounded-xl border border-[#E5E7EB] shadow-lg`. Title: "Add FAQ Entry" (in section header style). "Question" input with placeholder "What question will users ask?". "Answer" textarea with placeholder "Provide a clear, helpful answer...". Button row: "Save FAQ" (primary, loading: "Saving..." + disabled:opacity-50) + "Discard" (text button).

On save: call `createFaqItem(question, answer)`. On success: call `onCreated(item)`, close modal. On failure: show inline error.

Close on Escape key and backdrop click.
