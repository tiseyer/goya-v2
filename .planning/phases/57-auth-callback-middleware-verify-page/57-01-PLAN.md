Add import for `checkTrustedDevice` from `@/lib/device/checkTrustedDevice` at the top of the file.

After the existing `const { data: { user } } = await supabase.auth.getUser()` call (currently at line ~53), and BEFORE the role/invite handling block, insert the following device trust check:

```typescript
if (user) {
  const fingerprint = request.cookies.get('goya_device_fp')?.value
  if (fingerprint) {
    const trusted = await checkTrustedDevice(user.id, fingerprint)
    if (!trusted) {
      response.cookies.set('device_pending_verification', `${user.id}|${fingerprint}`, {
        httpOnly: true,
        sameSite: 'lax',
        path: '/',
        maxAge: 600,
        secure: process.env.NODE_ENV === 'production',
      })
      return NextResponse.redirect(new URL('/verify-device', origin))
    }
  }
  // If no fingerprint cookie present: proceed normally (first-visit edge case)
}
```

Important: this redirect must return a new response, NOT the existing `response` variable (which already has the session cookies set). Use `NextResponse.redirect(new URL('/verify-device', origin))` and set the cookie on that new response before returning it. The session cookies from `exchangeCodeForSession` are already set on `response`, so construct the device redirect as:

```typescript
const deviceRedirect = NextResponse.redirect(new URL('/verify-device', origin))
// Copy session cookies from response to deviceRedirect so user remains logged in
response.cookies.getAll().forEach(c => deviceRedirect.cookies.set(c.name, c.value))
deviceRedirect.cookies.set('device_pending_verification', `${user.id}|${fingerprint}`, {
  httpOnly: true,
  sameSite: 'lax',
  path: '/',
  maxAge: 600,
  secure: process.env.NODE_ENV === 'production',
})
return deviceRedirect
```

**Change 2 — Lock block (insert AFTER the password_reset_pending block, around line ~257):**

Define which routes are allowed while verification is pending:

```typescript
const DEVICE_VERIFICATION_ALLOWED = [
  '/verify-device',
  '/sign-in',
  '/sign-out',
  '/auth/callback',
]

// ─── Device pending verification lock ────────────────────────────────────────
if (devicePendingVerification && user) {
  const allowed =
    DEVICE_VERIFICATION_ALLOWED.some(p => pathname === p || pathname.startsWith(p + '/')) ||
    pathname.startsWith('/api/device-verification/')
  if (!allowed) {
    return NextResponse.redirect(new URL('/verify-device', request.url))
  }
}
if (devicePendingVerification && !user) {
  // Cookie exists but session expired — clear stale cookie and redirect to sign-in
  const clearResponse = NextResponse.redirect(new URL('/sign-in', request.url))
  clearResponse.cookies.set('device_pending_verification', '', { maxAge: 0, path: '/' })
  return clearResponse
}
```
